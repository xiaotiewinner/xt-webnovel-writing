# 反 AI 味清单（网文生成强制条款）

当前主流大模型生成的中文网文，即便流畅无语法错误，仍会被另一台大模型以 60%+ 置信度判为 "AI 生成"。原因不是单点错误，而是 18 个可检测的结构性指纹 + 若干隐蔽子条款。本文件列全 A–R 共 18 主条款 + 7 个子条款（B / E / G / H / N 的扩展或细化），共 **25 项**（另含 **A-补充**「好奇缝隙 / 章首抓眼」、**O-在场**「元叙事禁入」两处**并入主条款、不单独计数**的硬门），并对每一项给出：症状 → 典型 AI 示例 → 改写方向 → 检测信号 → 生成期规避规则。

子条款与 D / M / O / P / Q / **R** 等"元层级"主条款不是"锦上添花"，而是外部 LLM 在**多轮复查**以及本 skill 实测对比（人类写作样本 vs AI 生成样本）时抓出的结构性指纹：

- **D** 信息密度过干净（含闲笔密度 / 配角自主议题 / 废选项戏剧化三子维度）—— 决定"世界是否有自主生活"
- **M** 爽点链条过完整 —— 决定"剧情节奏是否留白"
- **O** 角色灵魂缺位 —— 决定"每个角色是不是主体"
- **P** 剧情算法化 / 想象力贫血（含怪异预算 / 废选项 / 延迟兑付 / 反套路检查）—— 决定"这章是不是 AI 能 10 秒预测"
- **Q** 转场机械 / 黏滞转场（含转场 5 类 / 禁用转场词 / 摩擦点必填）—— 决定"场景切换是 PPT 翻页还是有机延续"
- **R** 说明书式排除枚举 + 教程体微动作链 —— 决定"句子是不是像在写产品说明与操作步骤"

这六条任一不过关的叙事即便修辞通顺、对话自然、信息密度均匀，读者仍会在直觉上觉得"人都像纸做的 / 世界是假的 / 剧情可预测 / 转场很生硬 / **像在扫雷式读说明书**"。

任何生成类 skill（`webnovel-plot-design` / `webnovel-excitement-and-craft`）在**生成前**必须把本清单作为硬约束；`webnovel-pitfalls` 在**诊断时**把本清单当 Part B 逐条判定。

## A. 节拍器式推进

**症状**：全章像"爽文节拍器"——起阵 → 观察 → 点弱点 → 装逼对话 → 破局 → 反派震惊 → 奖励 → 升级 → 更大危机。每段一拍，无抖动，无偏离。

**典型 AI 示例（来自真实样本）**：整章 20+ 段，每段都精准承担一项功能。

**改写方向**：在每 3–5 段之间插入一次**节奏抖动**，三选一：
- 主角卡壳 / 失误 / 犹豫（哪怕一瞬间）
- 一段偏离主线的细节（环境、一段无关回忆、配角闲笔）
- 一段非功能性对话（不推剧情的闲话、嘴硬、嘟囔）

**检测信号**：章内任意连续 5 段都能被单独打上"功能标签"（破阵/装逼/夺宝/升级…）无遗漏。

**生成期规则**：
- 每章至少 **30% 段落必须是"无回报铺垫段"**：没有爽点、没有信息量、没有节点推进，只有状态或细节。
- 八步节奏表落到文本时，步与步之间必须留一次"呼吸"（一段非功能段）。

### A-补充 · 好奇缝隙与章首抓眼（并入 A，**不增加 25 项计数**）

**症状**：文笔「淡」——街景、气味、光线写了一圈，读者知道「在发生什么」，但**没有被勾住**，不会产生立刻想往下翻的问题链；章首像在写**环境说明书**而不是**事件切口**。

**典型 AI 示例**：整章前 500 字只有「冷 / 雾 / 路灯 / 窄巷 / 目光盯手机」匀速铺陈，没有非常规关系、没有非常规动作、没有「先抛钉子后解释」的缝隙。

**正面范式（高抓眼，非题材限定）**：
> 小姨子尖叫了一声给我踹出来。

读者瞬间积压问题：**谁**是小姨子？**踹谁**？**为什么尖叫**？看见了什么？配偶**当场什么反应**？——非常规人际称谓 + 非常规动作/声音 + **信息滞后（先不给解释）** 三件套。网文「抓人」往往靠这种**具体钉子 + 延迟解释**，不是靠形容词堆满。

**改写方向**：
- **章首前 ≈200 字内**必须有 ≥1 个**刺点钉子**（三选一即可，可叠加）：① 非常规人际称谓 / 关系暗示 ② 非常规动作或声音（与当下场景「不对付」更好）③ 非常规物件与处境并置。
- 全章每 **800–1200 字**须有 ≥1 处**好奇缝隙**：抛出**具体**信息但**至少滞后 3–8 行**才收一小步（不必当章兑完）。
- 「淡≠只有氛围」：氛围必须挂在**可被观察的小事件**上（一句带刺的话、一个多余的动作、一次误读），禁止连续多段纯「移动 + 泛泛情绪 + 环境」而无刺点。

**检测信号**：
- 章首 ≈200 字内无任何「关系 / 动作 / 物件」上的**非常规组合** → FAIL（必须重写章首）
- 连续 **4** 段可全部打上「纯氛围 / 纯位移 / 纯等待」标签 → WARN；连续 **≥ 6** 段 → **回滚级 FAIL**（退回 `webnovel-plot-design` 重排节奏与刺点）
- 全章「好奇缝隙」处数 < `max(2, ⌊全章字数 / 1200⌋)` → FAIL（短篇章按 2 处下限）

**生成期规则**：
- 写章首前自问：**如果删掉环境描写，还剩什么让读者必须看第二眼？** 若答不出 → 重写章首，先上钉子再铺街景。
- `chapter_meta.stats` 必填：`opening_hook_spike`（bool）、`curiosity_gap_markers`（int）、`flat_atmosphere_streak_max`（int）、`chapter_word_count`（用于阈值；可由 `chapter_body` 字数估算）。
- **新书首章特判（chapter 1 硬门）**：
  - 首章前 ≈500 字内必须满足：`series_opening_strike_count >= 2`（至少两类抓手并置：非常规关系/非常规动作声音/非常规物件处境）。
  - 首章必须 `opening_question_debt_present == true`（留下一个当章不完全兑付、但足够具体的追问债）。
  - 任一不满足都按 **回滚级 FAIL** 处理，不允许以“后面再热起来”豁免。

## B. 句式与语义复用

**症状**：高频句式在短篇幅内重复且几乎无变体。

**典型 AI 示例**：
- "XX 看着 YY。"
- "XX 没有回答。"
- "XX 的声音平静。"
- "XX 知道……才刚刚开始。"
- "但他不怕。"

**改写方向**：
- 同一句式（主谓结构 + 常用搭配）**同一场景内不得出现 3 次以上**。
- 第 2 次用变体，第 3 次必须换一个完全不同的表达方式（动作代替语言、侧面描写代替直述、让对方替他反应）。
- 主角、反派、配角各自持一套句式偏好，不共享。

**检测信号**：章内同一主谓句式命中 ≥ 3 次。

**生成期规则**：生成每一段前，查前文 500 字内是否出现过同句式；命中则强制换写。

### B-补充 · 词簇复读（并入 B，不增加 25 项计数）

**症状**：不是句式重复，而是同一类“气场词簇”高频回放（如掠食者/猩红/宿命/战栗/冷冽/深渊），导致文本像同一语料模板反复调用。

**典型 AI 示例**：同章多次出现“掠食者”“猩红”“命运的…/战栗”等抽象拔高词，且分布集中在转折段。

**改写方向**：
- 词簇去同构：同一语义场 500 字窗内最多保留 1 次，其余改为具象动作或场景证据。
- 把“抽象判断词”替换为“可拍摄细节”（动作、物件、声响、温度）。

**检测信号**：
- `lexeme_cluster_repeat_hits`（高频词簇复读命中）≥ 4 判 FAIL，≥ 7 判回滚级 FAIL。
- `abstract_aura_token_density_per_1k`（抽象气场词密度/千字）> 12 判 FAIL，> 18 判回滚级 FAIL。

**生成期规则**：
- 单章 `lexeme_cluster_repeat_hits ≤ 3`。
- `abstract_aura_token_density_per_1k ≤ 10`，超阈值必须以具象句替换至少 30%。

### B-补充2 · 近重复段落（并入 B，不增加 25 项计数）

**症状**：同一章内出现整段“同义替换式重写”并并排保留，常见于生成时改稿残留（同一开头句、同一信息链、仅替换少量词）。

**典型 AI 示例**：连续两段都以“下一个，林风。检测执事的声音平得像尺子量过。”起手，并重复“上台/众人漠然/测灵石像干涸河床”的信息序列。

**改写方向**：
- 同一叙事功能只保留一段终稿；替换稿必须删净。
- 若必须重复（仪式口令/咒语），每次重复必须带来新信息（结果、风险、关系至少一项变化）。

**检测信号**：
- `near_duplicate_paragraph_pairs`：近重复段落对数（段落级）
- `max_paragraph_similarity`：章内段落最大相似度（0~1）
- `duplicate_dialogue_openers`：重复对话开场命中（如“下一个，林风。”）

**生成期规则**：
- `near_duplicate_paragraph_pairs == 0` 才可 PASS；`>= 1` 为 FAIL，`>= 2` 为回滚级 FAIL。
- `max_paragraph_similarity < 0.88`；`>= 0.92` 直接回滚级 FAIL。
- `duplicate_dialogue_openers >= 1` 且无信息增量（结果/风险/关系未变）判 FAIL。

### B-补充3 · 比喻密度与同构句法（并入 B，不增加 25 项计数）

**症状**：比喻句（像/仿佛/如同）密度过高，且句法模板高度一致，形成“算法美感”而非角色视角。

**改写方向**：
- 每千字比喻控制在可感知但不过载区间；同一段内优先留 1 个比喻，其余改成动作/物件/声音证据。
- 同章避免反复使用同一壳句式（“X像Y”“像一块/一道/一阵…”）。

**检测信号**：
- `simile_density_per_1k`：每千字比喻数
- `simile_cluster_max`：连续比喻峰值
- `simile_pattern_repeat_hits`：同构比喻句法重复命中

**生成期规则**：
- `simile_density_per_1k ≤ 10`，`> 14` 判 FAIL。
- `simile_cluster_max ≤ 3`，`>= 4` 判 FAIL。
- `simile_pattern_repeat_hits ≤ 2`，`>= 3` 判 FAIL。

### B-补充4 · 感知数值化过度（并入 B，不增加 25 项计数）

**症状**：角色在无测量工具条件下持续输出“高精度数字感知”（如三度、零点五厘米、六十二度），文本像传感器日志而非人物体感。

**改写方向**：
- 无仪器/无专业流程场景，优先写“偏差感”“不适感”“经验估计”，避免伪精确小数。
- 每章保留少量“硬数字”即可，其余改为可拍摄细节（姿态、节奏、触感、声纹）。

**检测信号**：
- `hyper_precision_detail_hits`：无工具高精度细节命中（含小数、厘米级、角度级）
- `noninstrumental_numeric_density_per_1k`：无仪器数字密度/千字

**生成期规则**：
- `hyper_precision_detail_hits ≤ 1`，`>= 3` 判 FAIL。
- `noninstrumental_numeric_density_per_1k ≤ 6`，`> 10` 判 FAIL。

### B-扩展 · 主语单调与句长熵过低（隐蔽型）

比 B 主条款更深一层的问题：即便句式不重复，**主语与句长的宏观分布过于均匀**也是 AI 指纹。

**典型 AI 示例**：
> 他抬起剑。<br>
> 他闭上眼睛。<br>
> 他低头看着剑。<br>
> 他睁开眼睛。<br>
> 他没有回答。<br>
> 他抬起头。

表面每句不同，但每段都以 "他" 起头、都是 6–10 字短句、都是"主 + 谓 + 宾（短）"——节奏**匀速**。这正是外部 LLM 抓"AI 味"的关键信号：**"始终匀速输出"**。

**改写方向**：
- **主语轮替**：章内同一主语连续开头**不得 ≥ 3 段**。第 3 段起必须：
  - 省略主语（"剑锋抖了一下"）
  - 换视角主语（对方 / 道具 / 环境动作起头："剑鞘里无生剑自己响了一声"）
  - 用介词 / 时间副词 / 动作起头（"刹那间"、"剑影未落"、"风先到了"）
- **句长轮替**：章内连续 ≥ 4 段的段长差不得都 < 10 字。至少每 5 段出现一次长段（> 80 字）或复合句（包含 ≥ 2 个分句）。
- **节奏熵阈值**：本章主语开头出现频次 top-1 占比 ≤ 40%。超过 → 重写。

**检测信号**：
- 章内以 "他/她" 开头段落 ≥ 40%
- 章内 80% 以上段落是 ≤ 15 字的短主谓结构
- 连续 ≥ 4 段的段长标准差 < 8 字

**生成期规则**：
- 每写完 5 段 → 回看主语分布；top-1 占比 > 40% 时下一段强制换主语或无主语起头。
- 每写完 10 段 → 回看段长标准差；过低时下一段强制写 ≥ 100 字的长段（描写 / 心理 / 环境）。
- `webnovel-memory` 的 `state/used-patterns.md` 必须追踪上一章的主语分布与段长分布，作为本章禁用项输入。

## C. "第 X 剑" 节奏病

**症状**：单句成段连续堆叠制造节奏感——"第二剑。第三剑。第四剑。第五剑。"

**典型 AI 示例**：就是样本中的那段。这是 LLM 模仿网文节奏时的首选偷懒路径。

**改写方向**：
- 此手法一章**最多用一次**，且必须用在真正情绪/节奏顶点。
- 若要连续强化同一动作：穿插**不同层次描写**（外部动作→内部感受→敌方反应→环境回响），不是复读计数。
- "单句成段瀑布流"同样命中本条——即便不是"第 X 剑"的递增型，而是一问一答 / 一动一停被拆成 10+ 行单句独段，也是同一种 AI 节奏病。

**检测信号**：

| 维度 | PASS | WARN | FAIL（重写该段） | 回滚级 FAIL（退回全章） |
|---|---|---|---|---|
| 最长连续单句段数 | ≤ 2 | 3 | 4–5 | ≥ 6 |
| 连续 ≥ 3 的丛数 | 0 | 1–2 | 3 | ≥ 4 |
| 全章单句段占比 | ≤ 0.30 | 0.30–0.40 | 0.40–0.50 | > 0.50 |
| 全章单句段总数 / 段数 | — | — | — | **任一段数 ≥ 40 但单句段占比 > 0.45 直接回滚级** |

**生成期规则（强化）**：
- 禁止用纯数字/次数递增当节奏工具；必须配合"一次比一次"的变化维度（角度、速度、代价、对手反应）。
- 对白 / 心理段即便想制造停顿，连续单句成段**上限 2 段**。超过即在第 3 段位置插入一段 ≥ 2 句的复合段（动作 / 环境 / 心理任一）。
- **段落构成默认规则**：默认一段由 2–5 个句子组成；单句成段必须有**显式触发理由**（情绪顶点 / 打断 / 转场锚点 / 对话重击），不得"顺手就换行"。
- 写作时的本能纠错：如果连续 3 次按 Enter 都是短句，**第 3 次强制并到前一段**，再重新决定是否换行。
- `state/used-patterns.md` 追踪 `single_sentence_run_max` / `single_sentence_clusters` / `single_sentence_para_ratio`；任一指标命中 FAIL 档 → 当章拒收；命中回滚级 → 整章退回重写。

## D. 信息密度过于"干净" / 世界没有"自主生活"（回滚级）

**症状**：整段全是"推剧情 + 强化设定"，几乎零冗余——无天气、无旁人闲笔、无角色下意识小动作、无无关细节；所有配角只为主角说话、一切细节都将在 N 章内回收。世界像一个只为主角打开的布景，不像一个"主角恰好在里面生活"的真实场域。

**典型 AI 示例**：破阵、对话、夺宝、升级，每句都承担任务，没有一句"多余"；配角每一句台词都围绕主角；主角每一次选择都是"最优解"；所有伏笔都在当章回收。

**三个子维度（全部要过）**：

### D-1 · 闲笔密度（回滚级）
非功能性描写（天气 / 无意识小动作 / 无关物件 / 远处声音 / 身体状态 / 气味）**必须 ≥ 章字数 20%**，且**首次出现时绝不能带剧情解释**（"他摸了摸怀里的铜钱——那是师父给的" ✗；"他摸了摸怀里的铜钱。" ✓）。
- 数值：闲笔片段 ≥ **5 处 / 6000 字**；其中至少 **2 处与剧情完全无关**（删除后无任何后续引用）。
- **回滚级 FAIL**：闲笔 < 3 处 / 章 或 全部闲笔后续都被"剧情化回收" → 整章退回 `webnovel-plot-design`。

### D-2 · 配角自主议题（回滚级）
每章**至少 1 位非主角角色**必须有一段**与主角剧情无关的自主议题**（在聊别的事 / 在忙自己的事 / 在为和主角无关的事烦恼）。长度 ≥ 80 字或 ≥ 5 轮对话。
- 参考人类样本：食堂两个弟子整段八卦"真武堂骆北师兄被打成重伤"，和江北然完全无关。
- **回滚级 FAIL**：全章所有配角台词都围绕主角 → 退回 `webnovel-plot-design` 的人设与场景阶段。

### D-3 · 废选项戏剧化（回滚级，用于含"选择 / 系统 / 抽奖 / 面板"类设定的章节）
主角触发的选项 / 系统 / 抽奖 / 分支中，**至少一次主动选择最差 / 最无意义 / 最反直觉的那个**，并且通过该"废选择"带出人物性格或世界观细节。
- 正例（江北然）：连续 6 次选"路过"只得"力量+1"，带出"佛系怕事"的性格和系统重复触发的疲劳感。
- **回滚级 FAIL**：同类机制触发 ≥ 3 次而主角**全部选最优解** → 退回 `webnovel-plot-design` 的选择设计。

**检测信号**：
- 整章任意删掉一句都会影响剧情推进 → D-1 FAIL
- 全章配角台词 100% 围绕主角 → D-2 FAIL
- 面板 / 选择 / 系统触发次数 ≥ 3 全部最优选 → D-3 FAIL

**生成期规则**：
1. 每 300 字至少 1 句"剧情层面删掉也不影响"的非功能描写。
2. 动笔前为配角预先规划 1 个**独立议题**（自己的家事 / 自己的八卦 / 自己的烦恼），写进场景时让他们谈这个。
3. 选择类机制必须预留"反直觉选项"槽位，每 3 次触发内至少 1 次命中。

## E. 情感标签化

**症状**：用形容词列表标签化情绪——"震惊、恐惧、不甘、无奈"。

**典型 AI 示例**：
- 柳长风"眼神复杂。震惊、恐惧、不甘、无奈。"
- 「恐惧。愤怒。警惕。」—— 三个情绪词各自成单句，以句号并列
- 「**保护欲。**」 / 「**愤怒。**」 / 「**饥饿。**」—— 单个情绪词 + 单独成段 + 粗体

**改写方向**：
- 一个人物的情绪在同一段内**最多 1 个形容词**，其余必须用**具体生理/动作**表达。
- 可用信号：手攥紧到指节泛白、喉结上下滚一次、呼吸缺一拍、眼角一跳、后牙关咬住、嘴角扯了一下。
- 禁止"复杂"、"百感交集"、"五味杂陈"这种概括词。
- **禁止情感标签式呈现**（这是本条最隐蔽的变体，外部 LLM 第 4 轮复查高频抓到）：
  - 禁止「情绪词 A。情绪词 B。情绪词 C。」连续单词单句并列。
  - 禁止「**情绪词**。」单独成段 + 粗体——粗体越醒目，AI 味越重。
  - 禁止用"他/她感觉到了——" / "他/她意识到——"后紧跟单个情绪词独段的结构。
  - 情绪必须作为**某个动作 / 生理反应 / 画面的副产品**出现，不能作为独立的"情绪 token"被直接塞给读者。

**检测信号**：
- 单人物单段 ≥ 3 个并列情绪形容词 → FAIL
- 章内出现"情绪词 + 句号 + 单独成段"（有无粗体均算）：≥ 1 次 WARN，≥ 2 次 FAIL
- 章内出现"情绪词单独成段且粗体"（最重形式）：≥ 1 次即 FAIL

**生成期规则**：
- 情绪表达每用一次形容词，必须紧跟一个具体动作或生理反应作为"证据"。
- **情绪词禁止作为独立句子或独立段落出现**，必须嵌入叙述句之中。
- **粗体禁用于情绪词与主题句**（见 N-细化）。
- `state/used-patterns.md` 追踪 `emotion_token_hits`（情绪独句 / 独段数），本章 ≥ 2 → 立即重写该段。

### E-扩展 · 反应套餐化（强者震惊模板）

外部 LLM 常抓的点：即便每个反应都有"动作证据"，但**同类角色在同类情境下触发的反应是固定套餐**——强者被打脸一定用"脸色变了/渗出冷汗/胸口微起伏/沉默良久/说一句'不可能'"。**动作本身没问题，问题是组合永远一样**。

**典型 AI 示例**：
> 柳长风的脸色变了。<br>
> ……<br>
> 柳长风的额头渗出了冷汗。<br>
> ……<br>
> 柳长风站在光网外，胸口微微起伏。他没有说话。<br>
> ……<br>
> "不可能——"

一个反派从震惊到崩溃，全程走"脸色 → 冷汗 → 胸口 → 沉默 → 一句'不可能'"这套流水线。

**改写方向**：
- 每个有名有姓的配角/反派建卡时，必须固定 **≥ 2 条"只有他会有"的反应**：
  - 一种独特动作（咬指甲 / 抖袖口 / 用指关节敲桌 / 往左侧一步）
  - 一类独特语言（爱引典故 / 讲冷笑话 / 用某地方言 / 反问代替回答）
  - 一种独特生理反应（左眼跳 / 吞口水时发出响声 / 深呼吸前先闭眼）
- 震惊 / 愤怒 / 不甘等高频情绪，**禁止连用"脸色变了 + 冷汗 + 胸口 + 沉默"**这种标准套餐；至少替换掉 2 项为角色专属反应。
- 反派首次遭遇决定性打击时，必须出现一个**"不合时宜"的小反应**（想起别的事、笑了一下、做一个无关动作）——这是真人作者的标志。

**检测信号**：
- 反派被压制的段落里，出现 ≥ 3 项标准套餐词（脸色 / 冷汗 / 胸口 / 沉默 / "不可能"）
- 全章反派反应没有一条是"只有他会有"

**生成期规则**：
- 每个 `characters/<name>.md` 人物卡必填 `signature_reactions` 字段（至少 2 条）
- 生成反派反应段落时强制引用该字段；命中标准套餐 ≥ 3 项则替换 2 项为 signature

### E-扩展 2 · signature_reactions 明牌化（隐蔽型）

外部 LLM 第 4 轮抓到的隐蔽变体：作者给配角 / 女主设了"标志性小动作"（小指抖、耳垂红、摸右手腕），**本来是好事**，但每次出现都**明文指给读者看**——"他注意到她左耳垂红了"、"她左手小指又在抖"、"陈默看到她手腕上那道疤"。这种"作者怕读者漏掉伏笔"的标记行为，把一个本来真实的小动作变成了第二种"套餐"。

**症状**：
- 同一个 signature 反应（耳垂红 / 小指抖 / 手腕疤 等）在单章内被**明文指认**（叙述者 / 主角视角直接点出"我看到了 X"）≥ 2 次。
- 角色首次出现该反应时就被"指认 + 注解"（"她说谎的时候左耳垂会红"）。

**典型 AI 示例**：
> 陈默看到她左耳垂上的红。 …… 她的左手小指在抖。陈默注意到这一点。 …… 陈默看到了她左耳垂上的红。还没退。

三次出现，三次明牌化，等于作者拿荧光笔在伏笔上划线。

**改写方向**：
- signature_reactions 在单章内被**明文指认** ≤ 1 次（最多一次"看到 + 含义"）。
- 第二次及以后的出现必须**只写现象、不写解读**："她的小指停在杯沿上，停了半秒，又抬起来"（不写"陈默又注意到"、不写"她说谎时的习惯")。
- signature 解读的交代可以留白；读者反推得到的伏笔比明牌指认的伏笔更可信。

**检测信号**：
- 同一 signature 单章内明文指认 ≥ 2 次 → FAIL
- signature 首次出现即附带解读 / 含义注解 → WARN

**生成期规则**：
- `characters/<name>.md` 的 `signature_reactions` 必须同时记录 `明牌次数上限: 1`（全章计）。
- 每次写入 signature 行为时 self-check：本章是否已明牌过 1 次？已过 → 本次只写现象，不写指认语 / 解读。
- 上一章任一 signature 明牌 ≥ 2 → 下一章 LOAD 阶段该 signature 列为"本章只许现象级出现，禁止任何指认"。

### E-扩展 3 · 标志动作符号化过度（并入 E，不增加 25 项计数）

**症状**：角色“个性 tick”（如“啧”“扯嘴角”“眯眼”）在单章过度重复，尤其被固定在章首+章尾，形成机械标签而非自然行为。

**改写方向**：
- 同一 `signature_tick` 单章内出现 ≤ 2 次；第 3 次起必须替换为等价但不同的行为证据。
- 章首与章尾若使用同一 tick，需保证中段有“反向行为”或语气变化，不得纯粹首尾打点。

**检测信号**：
- `signature_tick_overuse_hits`：单章同一 tick 超阈命中
- `chapter_edge_tick_reuse`：章首与章尾同一 tick 复用（bool）

**生成期规则**：
- `signature_tick_overuse_hits >= 1` 判 FAIL。
- `chapter_edge_tick_reuse == true` 且近 3 章重复同一 tick 判 FAIL。

### E-扩展4 · 感情/隐晦情色过审风险（并入 E，不增加 25 项计数）

**症状**：为提升“暧昧张力”而滑向露骨表达或高风险关系表达，文本可读但平台审核风险高。

**改写方向**：
- 感情描写默认走“关系后果模式”：写边界、误读、克制、情绪余波，不写器官细节和具体性行为步骤。
- 隐晦情色只允许“动作前后反应”，禁止“教程式动作链”与体液/器官明示词。
- 任何高风险关系（未成年人、强迫/胁迫、权力滥用、血缘乱伦）一律降级为冲突事实，不得写成感官快感段。

**检测信号**：
- `suggestive_erotic_risk_hits`：隐晦情色风险命中（过密暗示/教程式性动作链）
- `explicit_sexual_content_hits`：露骨性描写命中（器官/体液/行为步骤）
- `high_risk_relationship_hits`：高风险关系命中（未成年人/强迫/权力滥用/血缘）
- `romance_target_ratio` / `erotic_tension_target_ratio` / `explicitness_target_ratio`：用户目标占比（开写前必问）

**生成期规则**：
- `explicit_sexual_content_hits > 0` → 回滚级 FAIL。
- `high_risk_relationship_hits > 0` → 回滚级 FAIL。
- `suggestive_erotic_risk_hits` 采用“占比驱动上限”：
  - `explicitness_target_ratio == 0`：`suggestive_erotic_risk_hits < 2`
  - `0 < explicitness_target_ratio ≤ 10%`：`suggestive_erotic_risk_hits ≤ 2`
  - `10% < explicitness_target_ratio ≤ 20%`：`suggestive_erotic_risk_hits ≤ 3`
- 禁止“过审清空”写法：若 `romance_target_ratio + erotic_tension_target_ratio > 0`，章节内必须出现至少 1 个关系张力触点（靠近/误读/克制反噬任一）。

### E-扩展5 · 感情不可控噪声缺位（并入 E，不增加 25 项计数）

**症状**：感情推进完全按“功能位正确答案”执行，角色反应稳定、可预测、无误判、无回摆，看起来像流程图。

**改写方向**：
- 每个关键感情场景必须加入至少一种“不可控噪声”：误读、嘴硬、反常回避、延迟回复、说反话后补救。
- 关系线必须允许短期回摆（靠近后后撤、承诺后迟疑），避免线性上升。
- 同一角色在同种刺激下应有轻微方差，不可永远同一反应脚本。

**检测信号**：
- `emotional_unpredictability_hits`：不可控噪声命中次数
- `affection_flow_reversal_count`：关系回摆次数（靠近->后撤）
- `emotion_response_variance_score`：同类刺激反应方差（0~1）
- `bilateral_dialogue_technique_hits`：全章“双方高完成度互相接住”次数（正确表达 + 正确接球 + 正确修复）
- `relationship_tacit_band`：关系默契档位（`low` / `mid` / `high`）

**生成期规则**：
- `emotional_unpredictability_hits >= 2`（单章至少两处噪声）。
- `affection_flow_reversal_count >= 1`（连续两章至少一处回摆）。
- `emotion_response_variance_score >= 0.3`，过低判 FAIL。
- 允许出现“完美接住”，但不得单极化：
  - `bilateral_dialogue_technique_hits >= 1`（全章至少 1 次，避免全程错频假噪声）。
  - `relationship_tacit_band = low` 时 `bilateral_dialogue_technique_hits ≤ 1`；
    `mid` 时 ≤ 2；`high` 时 ≤ 3。
  - 同时保留噪声与回摆约束：`emotional_unpredictability_hits >= 2`、`affection_flow_reversal_count >= 1`。
- 违反任一项即 FAIL（本项目标是“有接住也有不在同频”，而非单向禁用）。

## F. 能力 / 修为清单化

**症状**：能力/状态用清单式列出——"剑骨大成，无生剑在手。天剑诀，三卷合一。"

**典型 AI 示例**：
> "金丹中期。"（单独成段）
> "剑骨大成，配合天剑诀下卷，修炼速度恐怖如斯。"

**改写方向**：
- 修为/能力**禁止独段出现数字或等阶名**。
- 等阶必须嵌入一个**小动作 / 对比 / 被目击**的场景：
  - 他运气一试，灵力走一周天比三天前快了不止一倍（嵌入动作）
  - 旁边杂役弟子本能后退了半步（被目击）
  - 剑鞘里无生剑自己响了一声（道具反应）
- 新能力第一次用时必须**有代价或副作用**（气息紊乱、经脉发烫、一口血、视线发黑）。

**检测信号**：出现形如 "XX 境 / XX 期 / XX 大成" 且独段或连缀并列。

**生成期规则**：任何能力升阶都要用场景化呈现，不允许用清单或报菜名。

## G. 设定说明书化

**症状**：设定用"教的是——"、"功能是——"、"分为 A / B / C"的结构化标签直接输出。

**典型 AI 示例**：
> 天剑诀下卷，教的是——剑招变化。
> 天剑诀中卷，教的是——剑心锤炼。
> 天剑诀上卷，教的是——基础吐纳。
> 而终章，教的是——以血祭剑。

**改写方向**：
- 设定必须**嵌入体验**：主角翻开一卷，读到第三页皱眉；或配角说起传闻，主角对上一半。
- **长条目列表的设定禁止一次性出现**。三卷就拆成三个不同场景带出。
- 允许残缺、误读、矛盾——这些反而像真实叙事。

**检测信号**：连续 2 行以上 "XX——YY。XX——YY。" 的破折号对应式设定罗列。

**生成期规则**：任何"规则 / 功法 / 等级 / 阵法 / 流派 / 组织"介绍，禁止列表式输出；必须通过剧情事件、人物对白、主角体感带出，**一次只透露一条**。

### G-细化 · 角色当 PPT 直讲设定（首次亮相禁令）

**症状**：设定的名词 / 群体 / 体系**首次**在文中出现时，由一个角色用一段完整独白把它的**规模 + 性质 + 运作方式**一次讲全。即便没有用"是/不是/而是"模板（规避了 G-扩展 1），这依然是设定说明书的另一种形式。

**典型 AI 示例**（来自诊断样本）：
> "这个世界里有几千个像我一样的人。他们不穿披风，不喊口号，每天坐地铁、吃外卖、在朋友圈晒猫。但他们能在关键时刻——"
> ……
> "这个世界上，有几千个像我一样的人。"她说，"他们被称为——感知者。"

一次对话里塞了**群体规模 + 日常状态 + 能力特征 + 体系名**四件事。

**改写方向**：
- 设定名词（如"感知者"、"剑骨"、"天剑诀"、"七峰锁天阵"）**首次出现**时，必须满足下列三条件之一：
  - **体感式**：主角先体验到效果（看到别人看不到的东西、做到别人做不到的事），自己都不知道这叫什么；设定名词在下一章或更后由他人顺口带出。
  - **转述式**：路人 / 配角 / 敌方的闲聊 / 谩骂 / 怀疑里蹦出一次（"你莫不是个感知者？"），主角听到了但听不全。
  - **残缺式**：主角只知道一半（"我听说外面有一群'什么者'，具体干嘛的不清楚"），读者和他一起不懂。
- 设定名词**首次出现**的那一章，禁止该词同时携带以下任意两项信息：群体规模、组织架构、能力性质、宗派历史、生活状态、识别方式。**最多 1 项**。

**检测信号**：
- 设定专有名词首次出现时，同一对话段 / 同一叙述段内附带 ≥ 2 项结构性信息 → FAIL
- 同一角色单次发言包含 ≥ 2 个设定专有名词 → FAIL

**生成期规则**：
- `state/foreshadow.md` 或 `bible/` 的每条新设定必须标注 `first_reveal_mode: 体感|转述|残缺`；生成该章时强制按此模式展开。
- 对白中出现 ≥ 2 个设定专有名词 → 立即拆句，分到 ≥ 2 个场景。
- `state/used-patterns.md` 追踪 `setting_reveal_overload_hits`；上一章 ≥ 1 → 下一章设定首次亮相锁定为残缺式。

### G-扩展 1 · "是 / 不是 / 而是" 定义体

**症状**：用论文或教科书的定义结构输出设定——"X。它不是 A，不是 B，而是 C。"

**典型 AI 示例**：
> 剑骨。<br>
> 以剑为骨，以血为髓，以魂为意。<br>
> 它不是灵根，不是丹田，而是比两者更高级的存在。<br>
> 剑骨大成，意味着——<br>
> 他的灵力不再是灵力，而是剑气。

这是维基词条/课本口吻，不是小说口吻。

**被观察到的实际变体**（都属于同一类指纹，必须全部禁用）：

| 模板 | 样本 |
|---|---|
| 不是 A，是 B | 「不是雷声。是某种金属撞击的声音。」 |
| 不是 A——是 B | 「不是情绪变化——是语速变快了。」 |
| 不是那种 A——是 B | 「不是那种明显的抖——只是小指在微微颤。」 |
| 不是那种 A。是 B | 「不是那种'岁月静好'的平淡。是他觉得……」 |
| 不是 A。是某种 B | 「不是饿。是某种更原始的东西。」 |
| 不是对 A 的 X。是对 B 的 X | 「不是对林晚晴的恐惧。是对别的什么的恐惧。」 |
| 不是 A 的 X。是 B 的 X | 「不是他的愤怒。是尘土下面那个人的愤怒。」 |
| 不是 A 了。是 B 了 | 「不是语速变了。是温度变了。」 |
| X，意味着—— / 即 Y | 「剑骨大成，意味着——他的灵力不再是灵力，而是剑气。」 |

**以上 9 种模板全部视为同一条指纹的变体。以下改写方向必须全部满足**：
- 禁止任何"不是……是 / 而是 / ——是"对照结构。
- 禁止"X，意味着——"、"X，即——"、"X，也就是——"等破折号 / 冒号定义。
- 必须改为**体验式 / 他人转述 / 残缺式** 三选一：
  - 体验式：主角某次发力，旁人惊呼一句走了嘴的话，主角才意识到自己不同于普通修士
  - 他人转述：老者喝一杯茶，闲聊中丢一句"你这不是灵根，是剑骨"
  - 残缺式：主角只知道"自己的灵力比旁人重"，读者从战斗结果反推设定

**检测信号**：

| 档位 | 阈值（章内命中次数） |
|---|---|
| PASS | ≤ 1 |
| WARN | 2 |
| FAIL（重写） | 3–4 |
| 回滚级 FAIL（退回 plot-design 重写全章） | ≥ 5 |

本条是**命中率最高、最刺眼的 AI 指纹之一**。单章命中 ≥ 5 直接判整章 AI 生成，不给辩护空间。

**生成期规则**：
- prompt 里必须明文列出上表 9 种模板作为禁用清单。
- 生成每 500 字 self-check 一次，已命中次数写入 `state/used-patterns.md` 的 `definition_style_hits` 字段。
- 当章 `definition_style_hits ≥ 2` → 后续禁用该结构；`≥ 3` → 本章整段重写。
- 上一章 `definition_style_hits ≥ 2` → 下一章 LOAD 阶段该结构列为全章禁用词。

### G-扩展 2 · 三项并列打包讲解

**症状**：一个体系的 2–4 个部分被一次性打包讲完——"A 教的是 X。B 教的是 Y。C 教的是 Z。而终章，教的是 W。"

**典型 AI 示例**：
> 天剑诀下卷，教的是——剑招变化。<br>
> 天剑诀中卷，教的是——剑心锤炼。<br>
> 天剑诀上卷，教的是——基础吐纳。<br>
> 而终章，教的是——以血祭剑。

四行讲完四卷内容 = 数据库字段展开。

**改写方向**：
- 体系里的多项内容**禁止同章一次讲完**；至少拆到 3 章以上，由事件带出：
  - 下卷在本章由战斗结果自然反推（主角用完后回味"原来这就是剑招变化"）
  - 中卷放到下一个卡关点，主角读卷时只挑出半句带过
  - 上卷只在回忆或他人提及时一句带过
  - 终章"以血祭剑"必须留作**长线伏笔**，只在某人一句暗话里埋下，不直接说明作用
- 允许主角**理解错** / 只读懂一半——这反而真实。

**检测信号**：一章内对同一体系/组织/功法的并列部分 ≥ 3 项被连续讲述。

**生成期规则**：体系 / 典籍 / 地图 / 势力列表 ≥ 3 项时，强制分散到 ≥ 3 章，且每次透露**不超过一项 + 一句模糊暗示**。

### G-补充 · 系统提示模板腔（并入 G，不增加 25 项计数）

**症状**：系统提示文本像产品文案，模板重复且语义齐整（如 `【XXX：YYY——启动】`），脱离角色当下感知。

**典型 AI 示例**：
> 【好运过载：路径重构——启动】  
> 【检测到目标：唐小蛮（颜值85）——签到开启】

**改写方向**：
- 降低完整提示频次：完整提示只在关键节点出现，其余改为残片化、噪声化提示。
- 做角色化误读：同一功能的提示在不同场景改写成不同措辞，不允许机械复读。

**检测信号**：
- `system_prompt_template_hits`（同构提示模板命中）≥ 3 判 FAIL，≥ 5 判回滚级 FAIL。

**生成期规则**：
- 单章完整系统提示 ≤ 2 次；第 3 次起强制改为非模板残片提示。
- 禁止连续两次使用同一壳格式（`【名词：动词——状态】`）。

### G-补充 2 · 技术白皮书化（并入 G，不增加 25 项计数）

**症状**：技术段落像文档说明：术语密度过高、因果闭环过满、读者无需角色体验也能直接得到"正确机制"。

**典型 AI 示例**：同段连续出现 `API/节点同步延迟/热更新/重试机制/指数级` 等术语并完成完整解释链。

**改写方向**：
- 技术信息按"体验优先"暴露：先写主角误判与体感，再补一小段机制解释。
- 机制解释必须留缺口：每段最多交代一个关键机理，禁止同段讲完触发条件 + 计算过程 + 风控边界。

**检测信号**：
- `tech_jargon_density_per_1k`（每千字技术术语数）> 8 判 FAIL，> 12 判回滚级 FAIL。
- `tech_exposition_block_over_120`（>120 字技术说明段数）≥ 2 判 FAIL，≥ 3 判回滚级 FAIL。
- `tech_mechanism_closure_hits`（同段完整讲完触发→过程→结果）≥ 2 判 FAIL。

**生成期规则**：
- 单章技术说明长段（>120 字）最多 1 段。
- 技术解释段必须插入至少 1 条主角误判/失败代价，禁止纯说明书直讲。

### G-补充3 · 系统面板选项树模板化（并入 G，不增加 25 项计数）

**症状**：系统流段落连续出现“任务类型/奖励倍率/A-B-C-D选项/风险回报”完整面板，像游戏 UI 文档直接粘贴。

**改写方向**：
- 系统信息应碎片化、角色化：同一章仅保留 1 次完整面板，其余改为残片提示或后果反馈。
- A/B/C/D 选项若出现，必须伴随角色误读或情绪偏差，不可纯理性菜单。

**检测信号**：
- `system_option_matrix_hits`：完整选项矩阵命中次数（含 A/B/C/D 或等价多选结构）
- `bracket_system_block_count`：括号系统提示块数量（`【...】`）

**生成期规则**：
- `system_option_matrix_hits ≤ 1`，`>= 2` 判 FAIL。
- `bracket_system_block_count ≤ 4`，`>= 7` 判 FAIL。

## H. 战斗 = 最优解执行

**症状**：战斗是一道纯解题过程——找弱点 → 直奔弱点 → 一击破，无意外、无失误、无情绪干扰。

**典型 AI 示例**：
> 找阵眼 → 连续同点攻击 → 破阵。剑骨大成者的剑气，天生带有"破"的特性。每一剑都斩在同一个点上。

完美执行，零干扰。

**改写方向**：任何战斗/对抗至少植入**一次干扰变量**，三选一：
- **第一次尝试失败或不理想**（第一剑被吸收了一半；以为斩中弱点其实斩偏一寸）
- **敌方反应超预期**（对手本以为必死但反手打出一招没见过的东西）
- **主角自身波动**（看到某人/想起某事，剑出时手抖了一下）

并在战斗中**至少嵌入一处与战斗无关的感官**（风声、旁观者的某人的一声低呼、场边石柱倒下、远处一只惊起的鸟）。

**检测信号**：战斗全程主角动作无延迟、无失误、无情绪波动 → 典型 AI 战斗。

**生成期规则**：战斗大纲成型后，强制在高潮前插入一次"滞涩点"，并在结尾写一处战场之外的细节。

### H-扩展 · 算法式思维链（主角思考 = 解题）

比 H 主条款更深一层：即便战斗结果有失误，**主角的思考过程本身**也可能是算法化的——"建模 → 反馈 → 修正 → 收敛"的纯工程回路。

**典型 AI 示例**：
> 他感觉到了——阵法的灵力在增强，不再是之前那种缓慢压制的状态，而是变成了主动吞噬。<br>
> 剑光切进光网的时候，被灵力反震回来……<br>
> 他后退了一步。<br>
> ……<br>
> "叶无尘，你破阵靠的是精准。但七峰锁天阵不是静态的，它在吞噬你的灵力……"<br>
> ……<br>
> 他闭上眼睛，感受着体内剑骨的力量。<br>
> ……他只是感受着……<br>
> 他睁开眼睛。<br>
> 他看到了。<br>
> 光网最薄弱的地方，不在中心，而在——右下角。<br>
> 他动了。

每一步都是标准的"观察 → 发现问题 → 调整策略 → 收敛到正解"。这是工程流程图，不是文学叙事。

**改写方向**：主角的思考过程必须出现**至少一次"非理性噪声"**，五选一：
- **误判**：以为找到了正解，动手后才发现是错的
- **犹豫**：关键节点上卡住几拍，不是因为在思考，而是**走神**（想起一个无关的人/事/画面）
- **情绪干扰**：看到对方某个细节（白头发 / 旧伤 / 熟悉的动作），刀下意识慢了半拍
- **杂念**：战斗中突然想到一件毫不相关的小事（师兄借的玉简还没还、早饭没吃、某句话没问）
- **非最优选择**：明明有更稳的打法，偏偏选了更冒险/更笨/更出气的那种，理由说不清

**检测信号**：主角从"遇到阻碍"到"解决阻碍"的思考链条，全部可被浓缩成 "观察 → 判断 → 调整 → 成功" 的四步流程图，中间无任何跳出、走神、误判。

**生成期规则**：
- 任何 ≥ 300 字的主角思考片段，必须嵌入至少 1 条非理性噪声（见五选一）。
- 非理性噪声的出现不得总是"转化为正解"；至少 1/3 次必须**没有作用**、只是一下停顿或一点闲笔。

## I. 对话功能过载

**症状**：NPC 台词纯粹在替作者给读者讲设定或背景。

**典型 AI 示例**：
> "你忘了天剑宗立宗三千年的底蕴？"
> "困住你，让你交出天剑诀中卷和下卷。杀你——"

直接把立项设定倒进角色嘴里。

**改写方向**：
- 设定/背景信息**拆成多个场景、多个角色、多个碎片**，禁止一人一口气讲完。
- 每个有名有姓的角色必须有**一句"只有他会这么说"的口吻标记**：一个词偏好、一种语气、一种口头禅、一种省略习惯。
- 关键设定尽量用**行动/物件/环境**暗示，而非台词直述。

**检测信号**：一段独白式台词里出现 ≥ 2 个专有名词设定。

**生成期规则**：每段对话自查——"如果换另一个角色说，这句话会不会一字不差？" 若会，重写。

## J. 套路节点密度过高

**症状**：一章塞入太多类型节点——破阵 + 夺宝 + 升级 + 宗门政治 + 天下皆敌。每个节点独自成立都可以当一章高潮，堆在一章里就会显得"按爽文公式逐项执行"。

**典型 AI 示例**：样本章同时完成了 **一破阵 / 二夺宝（天剑诀 + 无生剑）/ 三升级（金丹初期 → 金丹中期）/ 四政治（柳长风联合其他宗门）/ 五天下皆敌**。

**改写方向**：
- **一章最多承担 2 个类型节点**（主节点 + 铺垫）。超过即拆章。
- 未写进本章的节点要以**伏笔**形式留着：一句暗示、一个眼神、一件小事——而不是一次性结算。
- 让读者在本章"吃到"一个爽点后有**情绪缓冲**，而不是继续赶场。

**检测信号**：一章内可独立打标签的类型节点 ≥ 4。

**生成期规则**：生成章纲时强制 count 类型节点；≥ 3 时预警，≥ 4 时拒绝生成，让用户决定保留哪两个。

## K 附加. 段落长度分布畸形

**症状**：整章 80%+ 段落为 ≤ 30 字的超短段，无长段，靠碎屏制造节奏感。尤其在对白 / 心理 / 转场段，容易变成"聊天记录式"一行一段的瀑布流。

**改写方向**：一章段长必须呈**阶梯状分布**：
- ≤ 60% 短句段（< 30 字）——承担节奏与冲击
- 20–30% 中段（30–80 字）——承担动作与对话
- **至少 3 段 > 80 字的长段**（其中至少 1 段 > 120 字）——承担描写、心理、环境、铺垫

**检测信号**：

| 档位 | 阈值 |
|---|---|
| PASS | > 80 字长段 ≥ 3，其中 ≥ 1 段 > 120 字；单句成段（≤ 15 字）占比 ≤ 40% |
| WARN | > 80 字长段 = 2 或 单句成段占比 40–55% |
| FAIL | > 80 字长段 ≤ 1 或 单句成段占比 > 55% |
| 回滚级 FAIL | 全章无任何 > 80 字段落 |

**生成期规则**：
- 生成完后统计段长分布；偏离阶梯 → 强制合并一部分短段或补写一段环境/心理长段直到达标。
- 长段不是凑字数，每段长段必须承担一项"非推剧情职能"：环境纵深、人物内心、无关回忆、场景声音景观、主角下意识联想。
- `state/used-patterns.md` 追踪 `long_paragraph_count`（> 80 字）与 `single_line_ratio`；未达阈值不允许 PERSIST。

### K-补充 · 场景块分段（与 C、Q 协同，**非新计数项**，并入 K 的生成期硬门）

**症状**：时间或地点已明显跳转（如「到家的时候是三点二十」「骑到十字路口」「红灯变绿」），却仍与上一段动作挤在同一段或仅用单换行粘连，读者像被拖着**瞬移**，同时加剧「一行一事」的 AI 感。

**改写方向**：
- **显著时空跳变**（跨 ≥ 30 分钟、或跨建筑 / 跨街区、或「晚→凌晨」类）→ **必须**使用 Markdown **空一行**起新段（段首可以是时间/地点锚一句，再跟叙述）。
- 同一场景内的细碎动作可并段；**一旦 POV 所处空间或叙事时间轴断裂**，必须先分段再写。
- 全章「该空行未空行」的硬切换（`k_scene_block_violations`）**= 0 为 PASS**；≥ 3 → FAIL；**≥ 5 → 回滚级 FAIL**（与 Q 的瞬移叠加时从严）。

**生成期规则**：`chapter_meta.stats.k_scene_block_violations` 必填；与 `transition_types` 对齐自查——每次 Q 桥切换后，若新场景首段仍黏在上段末尾 → 计 1 次违规。

## L 附加. 作者指纹缺位

**症状**：通篇风格统一、干净、无语气词、无口语、无情绪起伏——"工业冷静"，辨识度为零。

**改写方向**：在立书阶段为作品固定 **≥ 2 个"作者指纹"**，并贯穿全书：
- **常用动词偏好**（如总是用"挑"而不是"抬"、用"扔"而不是"放"）
- **一类反复出现的物象**（烟/酒/伞/旧书/某种花/某种光）
- **主角某个口头禅或标志性反应**（走神、哼一声、摸后颈、掰指节）
- **断句癖好**（偏爱逗号串 / 偏爱破折号 / 偏爱短问句）

**检测信号**：全章无任何可复述的"作者小怪癖"。

**生成期规则**：生成前从 `story-blueprint` 输出里读取 `author_fingerprint` 字段；若为空则补齐后再生成。`story-blueprint` 的人设步骤要同时产出作品级的作者指纹。

## M. 爽点链条过完整

**症状**：单章爽点严格按六步全走——**战斗升级 → 破阵 / 击败 → 反派震惊 / 认输 → 资源获取 → 连续升级 → 更大世界观展开**。每一步都刚好，没有一处"被打断 / 被延迟 / 被否定"。

**典型 AI 示例**（来自样本）：
> 破阵 → 柳长风"好" → 拿走天剑诀中下卷 + 无生剑 → 金丹初期 → 三天后金丹中期 → 四大宗门外部大冲突预警 → 叶无尘"不是防守，是进攻"。

全链条一气呵成 = "最优体验路径" = AI 的标志。

**改写方向**：爽点链条必须至少被**打断一次**，三选一（每章必选其一）：
- **延迟**：该爽的没有立刻爽到——赢了，但没拿到奖励；拿到了，但打不开；打开了，但此刻用不了
- **不给**：反派不认输 / 不交出 / 临走还羞辱一句；观众不买账；旁白者当面嘲讽主角做法
- **代价**：爽点必须以一个**无法补偿的损失**换到（身体残缺、信任破裂、一个重要 NPC 死、一个誓言被迫打破、主角自己失去某种东西）

必须**当章兑现损失或缺口**，不能只说"代价是什么，以后再说"——那是伪打断。

**检测信号**：章末对"战斗 / 夺宝 / 升级 / 反派收场 / 世界观升级"的 5 条检验全部 PASS 且无副作用、无延迟、无未收束 → 典型 AI 爽点流水线。

**生成期规则**：
- 生成本章章纲时，为每条爽点标注"打断点"（`delay` / `denied` / `cost` 三选一）
- 没标打断点 → 拒绝进入正文生成
- `webnovel-memory` 的 `state/used-excitement.md` 每条记录必须含 `interruption` 字段

## N. 质量曲线过稳定

**症状**："从头到尾维持同一质量曲线"——每句都写得差不多好，每段节奏都差不多稳，每个描写都差不多具象，没有一句特别出彩、也没有一句明显粗糙。整章像被一只手抹平过。

**外部 LLM 原话**：
> 这段文本"太稳定地优秀"，反而不像人。
> 人类写作的特征是：有好句，也有一般句；有节奏波动；有风格偏差。

**改写方向**：
- 每 2000–3000 字必须出现一次**"风格偏差"**，三选一：
  - **一句特别跳的金句 / 画面**（哪怕与上下文风格不完全贴合）
  - **一段明显"快写过去"的叙述**（不是漏洞，而是"作者这里懒了一下"——可以是直接总结、可以是跳过细节）
  - **一处意料之外的风格切换**（从冷硬到戏谑、从紧张到走神、从第三人称限知偶尔掺入一句主角的内心独白）
- 全章必须存在**至少 1 句明显不够好的句子**——不是为了差，而是为了**真**。
- 允许**不完美的过渡、不完美的收尾、不完美的情绪**；过度打磨反而暴露 AI。

**检测信号**：
- 全章每段的质量（具象度 / 画面感 / 情绪浓度）评分都落在同一狭窄区间
- 全章找不到任何一句"明显比平均水平差"的句子
- 全章也找不到任何一句"明显比平均水平亮"的金句

三者同时命中 → 质量曲线过稳定。

**生成期规则**：
- 每写完一段问自己："这一段和上一段的风格差异在哪？" 若答不出 → 下一段强制加偏差。
- 每章成稿后，agent 主动选 1 段**回退粗化**（把它改得直接一点、粗一点），再选 1 处**亮化**（写一句会让读者截图的句子），其余保留。
- `webnovel-memory` 的 `state/used-patterns.md` 追踪"段落质量方差"；方差过低 → 下一章补偏差强度。

### N-细化 · 粗体主题句点题（AI 怕读者没懂）

**症状**：全章隔几百字就出现一句被加粗的"主题总结句 / 内心结论句"，用来告诉读者"重点在这里"。这是 AI 写完后**怕读者没 get 到**的结构性冲动——粗体越多，AI 味越重。真正的老作家不会把主题句加粗。

**典型 AI 示例**（来自诊断样本）：
> **保护欲。**
> **那是他第一次觉醒。也是他第一次意识到，自己以为的"正常人生"，从始至终都是假的。**
> **他不能再装作不知道了。**
> **怎么面对一个自己完全不了解的恋人。**
> **"初感级的觉醒。"**
> **"比我预想的早了两个月。"**

一章六次加粗主题句 = 六次被 AI 按停、划重点。

**改写方向**：
- **全章粗体数 ≤ 1**，且粗体只能用于**非主题句的特殊文本**（如日记摘录、来信、系统提示、招牌字等物理文本），不得用于叙述者 / 主角的情绪 / 内心 / 主题总结。
- 禁止"粗体 + 独立成段 + 情绪 / 感悟 / 判断句"的三件套。
- 主题 / 情绪 / 领悟类句子一律**埋进正常叙述**，不加格式，不单独成段，不加省略号排比。

**检测信号**：
- 章内粗体数 ≥ 2 → WARN
- 章内粗体数 ≥ 3 → FAIL
- 章内粗体用于情绪 / 感悟 / 主题句 ≥ 1 → FAIL（无论总数多少）

**生成期规则**：
- 生成 prompt 明文禁止"粗体 + 主题句 / 情绪词 / 内心独白 / 人物心声粗体标注"。
- 成稿 self-check 统计所有粗体实例，分类为"物理文本类"（保留）与"主题 / 情绪类"（删除粗体或重写）。
- `state/used-patterns.md` 追踪 `bold_theme_hits`；上一章 ≥ 1 → 下一章全章禁用粗体。

### N-补充 · 金句收束与“太工整”尾句（并入 N，不增加 25 项计数）

**症状**：段尾/章尾高频出现“可摘抄金句”式总结（如“有些……不需要……”/“一旦……就再也……”），语义完整、情绪稳妥，但像写作课作业。

**改写方向**：
- 章尾优先交付“动作后果”而非“主题总结”。
- 若必须抒情，改成角色视角下的不完整句或带瑕疵的口语，不做万能结论。

**检测信号**：
- `golden_closing_line_hits`：金句式收束命中
- `maxim_style_summary_hits`：格言体总结句命中

**生成期规则**：
- 单章 `golden_closing_line_hits ≤ 1`；章尾最后 120 字出现格言体总结则计重权重。
- `maxim_style_summary_hits >= 2` 判 FAIL。

## O. 角色灵魂缺位 / 功能性存在

**症状**：每个出场角色（含配角、反派、路人、师长、动物、灵兽、法宝拟人）只承担**剧情功能**——推进主线、反衬主角、传信、送资源、衬托反派——却没有**独立于本场景功能的内部状态**。读者能一眼回答"这个角色在故事里做什么"，但回答不出"ta 是谁 / ta 自己想要什么 / ta 为什么这样"。外部 LLM 第 4 轮复查原话（复述）："配角、反派、动物更像剧情道具而非人，整本书像一个人在跟很多 NPC 说话"。

这是比 E-扩展（反应套餐化）和 I（对话功能过载）更深一层的指纹。那两条是"怎么反应 / 怎么说话"的表层，O 盯的是**主体性本身是否存在**。

**典型 AI 示例**：
- 反派出场 = 傲慢 / 恨主角 / 要资源；离场 = 震惊 / 失败。**没有第三维度**。
- 女配出场 = 仰慕主角 / 需要主角救 / 暗中帮主角；全部动作都围绕主角展开，仿佛她没有主角之外的生活。
- 灵兽 / 通灵动物 = 只在关键时刻预警 / 救主 / 点头摇头回应指令，活得像会动的物品，无独立偏好。
- 师父出场 = 传功 + 说教 + 转身走开；没有自己的近况、情绪漂移、私人麻烦，也没有和主角无关的时刻。
- 路人甲的功能就是"证人" / "报信人"——他的台词可以换成任何路人乙且毫无差异。

**改写方向**：每个符合下列之一的角色都必须在其人物卡与当章正文里携带灵魂字段（**与本章剧情功能无关的内部状态**）：

| 角色级别 | 判定条件 | soul_fields 最低条数 | 首次登场章灵魂渗透要求 |
|---|---|---|---|
| 关键角色 | 主角 / 当章 POV / 本书预设 top 5 配角 / 核心反派 / 女主或男主 | **≥ 2 条** | **首次登场即必须有 ≥ 1 处非功能灵魂渗透**，不因出场次数豁免 |
| 重要有名角色 | 已有人物卡、且当章出场 ≥ 2 次 | ≥ 1 条 | 当章至少 1 处灵魂渗透 |
| 次要有名角色 | 有名字、出场 = 1 次但非纯路人（有名有台词） | ≥ 1 条（可留白续写时补） | 当章如出场，至少 1 处"只属于他"的不可替换细节 |
| 纯路人 NPC | 无名，单场景工具性出现 | 免 | 至少有一处可识别的**不可互换**特征（口吻 / 动作 / 穿着 / 所处状态任一），不得是"功能模板路人"|

"关键角色首次登场即生效"是关键修正——不再允许"这是第一次登场，以后再显灵魂"的拖延。第一章登场的女主 / 核心配角必须在**第一章就有灵魂渗透**，否则回滚到 story-blueprint 补 soul_fields 再重写本章。

**soul_fields 四类**（从以下至少取所需条数）：

1. **core_wound（核心创伤 / 遗憾 / 执念）**：一条影响当下行动但**不直接与主角相关**的旧伤——死去的师妹、没能带回家的人、一次无法挽回的选择。
2. **private_desire（私人欲望）**：一条对剧情推进**无用**的私愿——想攒钱去看某座山、想见一个已死的人、想戒烟但戒不掉、想有一天再吃一口童年那种点心。
3. **contradictory_belief（不合身份的私下信念）**：与其阵营 / 职位 / 立场**不一致**的私下态度——正道执法堂长老暗地觉得魔道那人说得对；反派私下敬佩自己亲手杀掉的老者。
4. **unreasonable_preference（非理性偏好 / 怪癖）**：怕打雷、讨厌某种食物因为一段童年、只用左手系绳、听到某种乐器会走神。不需要解释，不需要伏笔回收。

**动物 / 灵兽 / 灵物**必须有**独立反应坐标轴**，至少一条：
- 非本能偏好（只听某种乐器声会停下 / 只挑某种草吃 / 怕某种颜色）
- 与主角不同步的情绪（主角紧张它却漠然；主角高兴它却焦躁）
- 私人行动线（每日固定时辰去某处蹲一会儿，与剧情完全无关）
- **禁止**单纯"关键时刻预警 / 救主 / 回应指令"的工具模式——至少要在非关键时刻有一次与主角意图无关的独立反应。

**反派**不能只有"对主角的恨"或"对权力的渴望"。必须有**与主角无关的第三维度**：老伤发作、每月初一要去祭拜某人、极度怕疼、有一段羞于启齿的出身、会在深夜下意识哼一句他本人都已忘了出处的曲子。第三维度不必解释、也不必与剧情兑现，**它只是被偶然看见**。

**检测信号**：
- 任一关键角色的人物卡 `soul_fields` 少于 2 条非空 → 回滚级 FAIL
- 任一关键角色首次登场章无灵魂渗透 → 回滚级 FAIL（退回 story-blueprint）
- 任一出场 ≥ 2 次的有名角色该章无灵魂渗透 → FAIL
- 任一角色的所有出场段落功能标签都是"推进 / 反衬主角 / 送信息 / 送资源" → FAIL
- 动物 / 灵兽 100% 服务于主角意图、全章无任何非功能反应 → FAIL
- 反派所有行动 / 台词都可折叠为"想赢"或"恨主角"两项，找不出第三维度 → FAIL
- 任一配角台词"换成另一个同阵营同身份的角色来说也毫无违和"（与 I 联动的强化信号）→ FAIL

**生成期规则**：
- 每个关键角色的 `characters/<name>.md` 的 `soul_fields` 字段 **≥ 2 条非空**；每个重要角色 **≥ 1 条非空**；空 → `webnovel-memory · PERSIST` 拒收本章并回滚到 `webnovel-story-blueprint · STEP 2` 补齐。
- 生成当章前，先加载该章所有以下角色的人物卡：
  - 当章出场 ≥ 2 次的有名角色
  - **当章首次登场的关键角色（无论出场几次）**
  - 连续 2 章灵魂渗透缺位的任一角色
- 将其 `soul_fields` 作为"必现清单"嵌入 prompt：
  - 必现：本章至少 **1 处**让该条灵魂字段**非功能性地渗出一次**（不推动剧情，只做存在确认）。
  - 渗出形式示例："柳长风动手前揉了揉右手腕——那是早年一场他从未对人提起过的旧伤。" / "师父说话时眼神飘到了窗外那盆枯掉的兰花，又收回来。"
  - **渗出必须与当章剧情功能解耦**：把该句删掉剧情推进不受影响，才算数。
- 动物 / 灵兽段落：若整章出现 **100%** 服务于主角指令 → 下一章强制补 1 处"与主角指令无关的独立行动 / 偏好"。
- 反派章节自检：若该反派当章所有动机都可折叠为"想赢 / 恨主角" → 重写，补第三维度；重写 2 轮仍命中 → 交付时列入 FAIL 项告知用户。
- `webnovel-memory` 的 `state/used-patterns.md` 追踪 `soul_bleed_log`（每个有名角色每次出场是否有至少 1 处灵魂渗透）+ `character_interchangeability_check`（配角台词是否可互换）；任一角色连续 2 章缺位 → 下一章升级为**回滚级硬门**。

### O-在场 · 元叙事禁入（并入 O，**不增加 25 项计数**）

**症状**：叙事声音或人物内心使用**书籍体外坐标**——仿佛角色知道自己在「分章连载」里。

**典型 AI 示例（来自本 skill 实测）**：
> 「基础吐纳法」他刚才在**上一章**已经看过一次了。

读者知道「上一章」是什么；**书中人物**没有「章」的概念，只能说「昨天夜里 / 刚才在路上 / 前一会儿 / 上次点开时」等**故事内时间**。

**改写方向**：
- 全文（叙述 / 对白 / 自由间接引语式的内心）禁止使用：`上一章` `下一章` `本章` `上回书` `各位读者` `作者` `弹幕` `评论区` 等暴露**册页结构或读者存在**的用语。（例外：用户明确要求写 meta 小说 / 打破第四墙，且 prompt 中已声明豁免条款。）
- 把「上一章已经发生过的信息」改写成人物的自然回忆或口癖：**「他前一阵刚扫过一眼那几行字」** 而不是 **「上一章看过」**。

**检测信号**：
- 正文命中禁表用语 ≥ 1 次 → **回滚级 FAIL**（整章退回 `webnovel-plot-design` 做语言层清零）

**生成期规则**：
- 成稿后全文检索禁表词；命中即删改，不得留「作者开玩笑式」的残留。
- `chapter_meta.stats.meta_language_hits` 必须 = 0 才允许 PERSIST。

## P. 剧情算法化 / 想象力贫血（回滚级）

**症状**：剧情走"最优解算法"——重生 → 发现金手指 → 立刻使用 → 立刻收益 → 立刻规划未来；所有冲突当章回收；每个伏笔当章兑付；每个选项都是"最优"；场景、设定、节奏全都落在最常见的网文模板上，任何一点被 ChatGPT 问到"你觉得接下来会发生什么"都能被 80% 命中。

**典型 AI 示例**：
- "重生到高三 → 发现签到系统 → 第一次签到拿到灵石 → 规划明日签到地点" —— 一个断点都没有，全程线性最优
- 女主第一次登场就下定决心 "我要保护他"；反派第一次登场就宣告目标
- 每个设定登场即解释，每个伏笔登场即回收

**改写方向（"反算法化"四板斧）**：

### P-1 · 怪异预算（Weirdness Budget，回滚级硬门）
每章**必须 ≥ 1 处"剧情无法吸收"的设定 / 场景 / 细节**——它不是伏笔，不会回收，只是因为这个世界就这样。
- 正例：武侠门派里有个弟子只爱喝冰美式；修仙宗门食堂主打"翡翠白玉套餐"；破败物流仓墙上有只画歪了的柯基头像。
- 写作时的自检问句：**"这个细节三章之后会被回收吗？"** 如果答"会" → 它不是怪异预算，得再加一个。
- **回滚级 FAIL**：全章找不到任何无法回收的怪异元素 → 退回 `webnovel-story-blueprint` 的世界观与场景设计阶段。

### P-2 · 废选项 / 反直觉选择（对应 D-3 协同）
主角、配角、反派至少一方在本章做 **≥ 1 个"明显不是最优"的选择**，且该选择是**性格驱动**的（不是为了卖惨 / 不是为了剧情需要）。
- 正例：江北然连续 6 次选"路过"；林晚晴明知道说谎耳垂会红还是要说。
- **FAIL**：主角所有行动都可被 ChatGPT 在 10 秒内预测。

### P-3 · 延迟兑付（Setup without Payoff）
每章至少埋 **1 个不会在 5 章内回收的伏笔 / 细节**；且不允许"章末暗示性独白"提示读者这是伏笔（例：×"他不知道的是——林晚晴此刻想的是……"）。
- 正例：师父眼神飘到窗外那盆枯兰花；主角手腕内侧一道从未被人提过的旧疤。
- **FAIL**：本章所有设定都在本章内被动用或被解释。

### P-4 · 反套路检查（Anti-Trope Audit）
写作前必须生成 **5 个"最常见的下一步"清单**，然后**明令禁止使用前 3 个**；真正写的那一步必须是清单第 4 名之后或根本不在清单里的。
- 落地模板（强制在生成前执行）：
  ```
  我是 AI，面对当前场景，我能想到的最常见接续是：
  1. ______ （禁用）
  2. ______ （禁用）
  3. ______ （禁用）
  4. ______ （候选）
  5. ______ （候选）
  真正写：____________（必须 ≠ 1–3 且能体现主角性格）
  ```
- **回滚级 FAIL**：本章关键转折落在"最常见的三个"中任何一个 → 退回 `webnovel-plot-design` 的情节设计阶段。

**检测信号**：
- 全章冲突数 ≥ 3 且**全部在本章解决** → FAIL
- 全章伏笔数 ≥ 3 且**全部在本章兑付** → FAIL
- 本章"最大亮点"可被 "重生/系统/扮猪吃虎/反派震惊" 等标准网文套路词完全概括 → P-4 FAIL
- 全章无任何"说不上有什么用"的元素 → P-1 回滚级 FAIL

**生成期规则**：
1. 动笔前必须完成 P-4 的 5-清单并落盘到 `state/anti-trope-log.md`（每章一条）。
2. 写完后做一次 `deletion_test`：随便删掉 1 个"看似只是氛围"的细节，看后面剧情是否仍然通顺——通顺 = D/P-3 合规，不通顺 = 该细节其实是伏笔，不是怪异预算。

### P-补充 · 巧合闭环过快（并入 P，不增加 25 项计数）

**症状**：关键推进连续由"刚好"触发（刚好摔、刚好捡、刚好被引路、刚好遇见），单章形成高效率命运闭环，像流程图而非叙事。

**典型 AI 示例**：A 事故 → 捡到神秘物件 → 被物件引路 → 立刻遇见关键角色，且中间没有主角主动代价。

**改写方向**：
- 每 2 次偶然触发后必须插入 1 次主动选择，并给出代价（时间/社交/资源）。
- 把闭环拆章：本章只完成触发与误判，会合或揭示延后。

**检测信号**：
- `coincidence_chain_hits`（连续偶然驱动节点数）≥ 4 判 FAIL，≥ 6 判回滚级 FAIL。
- `forced_detour_hits`（主角可回头但被叙事强行导向单一路径）≥ 2 判 FAIL。

**生成期规则**：
- 单章 `coincidence_chain_hits ≤ 3`；达到 3 后后续推进必须转为主动决策。
- 禁止同章出现"偶然捡物 + 偶然引路 + 偶然会合"三连闭环。

### P-补充2 · 文化 shorthand 贴脸对抗 + 收束节拍（并入 P，**不增加 25 项计数**）

**症状**：辞藻华美但想象力扁平——能力、身份、对抗全靠原创生造词解释；读者没有「意外贴脸」的认知裂缝。

**好示例（读者共有符号 × 具象对抗 × 收束刹车）**：
> 沈屹飞用枪泯灭了一道门，证明了他是【成吉思汗】。  
> 杨教授开「门神壁垒」，五阶，连沈屹飞的枪都泯不了——他当然也是真的【尉迟恭】。  
> ……「【李白】就不用展示了。」

**改写方向**：
- **贴脸对抗**：把读者共有的文化 / 历史 / 典故 / 俗语（不必用【】框）**钉进一次具体对抗动作**里——符号为动作服务，而非旁白解释世界观。
- **收束节拍**：在「预期已被抬高」之后，允许一次**拒展示 / 留白 / 一句带过**打断叠宝具，形成喜剧或冷幽默的信息控制（如上「李白就不用展示了」）。
- **反差**：用「阶位 / 资历 / 身份」与「现场表现」错位制造裂缝（例：刚进阶却打出老兵油子精度），再用旁人反应钉死。

**检测信号**：
- `cultural_shorthand_clash_hits`：全章「共有文化符号 + 对抗动作」并置次数；**正文章 = 0 → 回滚级 FAIL**。
- `withhold_beat_present`：是否出现「抬高预期 → 拒展示 / 留白收束」节拍；**= false → FAIL（补一句收束才可 PERSIST）**。

**生成期规则**：
- `draft_prose` 每章必须 `cultural_shorthand_clash_hits ≥ 1`（符号可换域：历史 / 戏曲 / 诗词 / 江湖切口 / 网络梗——但必须与对抗动作同场并置，禁止纯标题式点名）。
- 每章必须 `withhold_beat_present == true`（至少一处收束刹车；可与对白合一）。
- `chapter_meta.stats` 必须填报上述两项；`used-patterns.md` 滑窗追踪，连续 2 章 `cultural_shorthand_clash_hits == 0` → 下一章升级为**回滚级硬门**。

### P-补充3 · 模板剧情链条过密（并入 P，不增加 25 项计数）

**症状**：单章关键推进过度贴合“黄金三章模板链”（如：测灵失败 → 执事压迫 → 离宗受辱 → 神秘老人/道具 → 金手指显影），读者可在中段高命中预测后续。

**改写方向**：
- 同章模板链条节点不得连成“直线五连”；至少插入 1 个与主线收益无关的扰动节点（误判/岔路/关系反转）。
- “神秘老人 + 神秘道具 + 能力命名”三项禁止同章全开，至少拆开一项到下一章。

**检测信号**：
- `trope_chain_hits`：模板节点命中总数
- `trope_chain_max_run`：连续模板节点最长链
- `predictability_score`：low/med/high（基于链条密度与是否缺扰动）

**生成期规则**：
- `trope_chain_hits >= 4` 判 FAIL。
- `trope_chain_max_run >= 3` 且 `weirdness_budget_count == 0` 判回滚级 FAIL。
- `predictability_score == high` 时必须重排章节节拍并补“非收益扰动节点”。

### P-补充4 · 跨题材缝合过载（并入 P，不增加 25 项计数）

**症状**：同章同时高密度拼接职场/修真/系统/悬疑等多赛道爽点，读者能看出“要素清单式堆叠”。

**改写方向**：
- 首章只允许 1 条主赛道 + 1 条副赛道强曝光，其余以弱信号埋伏，不得同章全开。
- 每新增一条赛道，必须付出“信息负担代价”（删掉一段解释或延后一个爽点）。

**检测信号**：
- `multi_genre_graft_count`：同章强曝光题材轨道数
- `graft_overload_hits`：无代价硬缝合命中

**生成期规则**：
- `multi_genre_graft_count ≤ 2`（首章更严格），`>= 3` 判 FAIL。
- `graft_overload_hits >= 1` 判 FAIL；首章命中按回滚级处理。

### P-补充5 · 伏笔装载过密（并入 P，不增加 25 项计数）

**症状**：短篇幅内同时投放过多高权重线索（身世债、旧伤、异象、关键道具、异常动物、神秘联系人等），读者感到“每句都在埋点”。

**改写方向**：
- 单章只保留“主线伏笔 + 一条侧线伏笔”强曝光，其余改弱信号或后移。
- 伏笔必须有轻重层级：至少 1 条可暂时作废，避免“条条都要回收”。

**检测信号**：
- `foreshadow_pack_density_per_1k`：千字高权重伏笔数
- `high_priority_foreshadow_count`：高优先伏笔数

**生成期规则**：
- `foreshadow_pack_density_per_1k ≤ 3`，`> 5` 判 FAIL。
- `high_priority_foreshadow_count ≤ 2`，`>= 4` 判 FAIL。

### P-补充6 · 反差吊胃口（低频可选机制，并入 P，不增加 25 项计数）

**定位**：这是“可选钩子”，不是默认模板。用于少数章节制造“认知翻面”张力（如敌意角色短时反常协助、冷硬角色露出不合设定软肋），但必须低频、可解释、可回收。

**启用前提**：
- 本章存在“关系单线化”风险（冲突太直、情绪太平）时才可启用。
- `contrast_hook_frequency_10ch ≤ 2` 且 `contrast_hook_chapter_gap ≥ 3`（10 章内最多 2 次，且至少隔 3 章）。
- 若上一章已启用反差钩子，本章默认禁用（冷却）。

**检测信号**：
- `contrast_hook_enabled`：是否启用本机制（bool）
- `contrast_hook_frequency_10ch`：近 10 章启用次数
- `contrast_hook_chapter_gap`：距上次启用章距
- `contrast_hook_misuse_hits`：滥用命中（无前置动因/无后果/无回收）

**生成期规则**：
- 默认 `contrast_hook_enabled == false`；未满足前提不得开启。
- 开启时必须同时给出：`触发动因` + `反差动作` + `关系余波` 三件套；缺任一计 `contrast_hook_misuse_hits += 1`。
- `contrast_hook_misuse_hits >= 1` 判 FAIL；`contrast_hook_frequency_10ch > 2` 判 FAIL。

## Q. 转场机械 / 黏滞转场（回滚级）

**症状**：场景切换靠"然后"、"接着"、"就在这时"、"三天后"、"第二天"；时间跳跃靠旁白；人物位移靠"他走到了 X"；两个场景之间**没有摩擦**（没有延迟、没有打断、没有一个小意外）。读起来像 PPT 翻页：上一页讲完，翻到下一页。

**典型 AI 示例**：
> 他站起来，走到湖边。  
> **就在这时**，一个小师妹在哭。  
> 他转身离开。  
> **第二天**，他来到了食堂。

**禁用转场词（零容忍，命中即 FAIL）**：
> 就在这时 / 与此同时 / 然而就在 / 就在他以为 / 三天后 / 第二天 / 一个星期后 / 半个月过去 / 时间一晃 / 转眼间

（独立使用禁止；若嵌在对话或人物心理活动中，需在同场景内被**反讽 / 打断 / 二次否定**才允许。）

**转场 5 类（每次场景切换必须落入其中一类）**：

### Q-1 · 感官桥（Sensory Bridge）
上一场景的**一个感官元素**延续到下一场景，再被新场景的另一个感官替换掉。
- 例："饭团的热气已经散了……" → 下场景开头："便利店门口的玻璃映着他湿了一半的袖口" —— 触觉 → 视觉延续。

### Q-2 · 物件桥（Object Bridge）
一件**具体的物件**从 A 场景带到 B 场景，且在 B 场景发挥**与 A 不同的**作用。
- 例：矿泉水在便利店拧瓶盖 → 在仓库拧瓶盖 → 在家里拧瓶盖（同一动作三次功能递进）

### Q-3 · 对话打断桥（Interruption Bridge）
用**对话中途被打断**实现切换：上一段最后一句对话没说完，下一段开头是另一段新对话开始。
- 例：上段 "...我想跟你说——" / 下段 "...你终于来了。"

### Q-4 · 摩擦点桥（Friction Bridge）
切换过程中必须有**一个小意外 / 小延迟 / 小错位**。
- 例：去食堂路上踩到一摊水；公交车司机笑得奇怪；导航显示"已到达"但面前是荒地。
- **这是摩擦点必填的硬规则**：任何超过 5 分钟的场景跨度，必须有 ≥ 1 个摩擦点作为桥。

### Q-5 · 情绪错位桥（Emotional Dissonance Bridge）
上一场景情绪 A，下一场景环境/配角气氛**不匹配该情绪**，通过错位制造张力。
- 例：刚在仓库目睹异象 → 下车时公交司机对他咧嘴笑。

**检测信号**：
- 章内任何一次场景切换**不命中 5 类中任何一类** → FAIL
- 使用禁用转场词 ≥ 2 次 → 回滚级 FAIL
- 两个相邻场景之间**主角是"瞬移"**（上段结束在 A，下段开头直接在 B，中间无任何感官/物件/对话/摩擦/情绪桥） → 回滚级 FAIL

**生成期规则**：
1. 生成每个新场景前，先声明：**"本次转场使用 Q-? 桥，锚点是 ________"**，写入草稿注释。
2. 若无法声明，则该场景禁止开始，先回前一场景补摩擦点或感官延续。
3. `webnovel-memory` 的 `state/used-patterns.md` 追踪每章 `transition_types`（5 类命中分布）；**单一类型使用 ≥ 3 次 → 下一章禁用该类型**。

## R. 说明书式排除枚举 + 教程体微动作链（**回滚级**）

**症状**：叙事句法像在给读者做**产品排除**与**操作验收**——先列「不是什么」，再揭晓「是什么」；动作像软件教程逐步点击，一步一屏，无人物学、无走神、无废步。

**典型 AI 示例（来自本 skill 实测样本）**：
> 手机屏幕突然亮了。不是来电，不是短信，是某个他不记得安装过的应用自己弹出来一个界面——全黑背景，中间一行白字：……  
> 他把手机倒过来晃了晃，又点了几下屏幕。界面没关，白字还在。  
> 烟盒是压扁的，只剩最后三根。他叼了一根出来，又塞回去，再掏出来，打火机咔哒了两下，火苗被风一吹就灭了。第三次才点着。

**为什么这是 AI 指纹**：
- **R-1 排除式三联 / 目录句**：用「不是 A，不是 B，是 C」「既非…也非…而是…」「没有…也没有…只剩…」把认知打包成**对照表**。人类在同一场景更常**直接写现象**或只保留**一个**否定（「不是外卖软件，界面却像外卖弹窗那样弹出来」），很少连续两次否定再接收束。
- **R-2 教程体微动作链**：同一情绪节拍内，**≥ 4 个**无对白、无心理、无环境的纯机械动词短语排成串（掏出→又塞回→再掏出→咔哒两下→第三次才…），像在写「步骤 1–5」。读者感到的是**操作手册**，不是人在场。
- **R-3 验收式短句对**：动作失败后跟两句**碎片验收**（「界面没关。」「白字还在。」）强化「状态机读数」感，与 C 的单句段瀑布叠加时尤其致命。

**改写方向**：
- **R-1**：删掉双重否定，只写**一个**意外源 + 感官；例：「屏幕自己亮了，锁屏界面上浮着一块黑，像有人把通知栏撕走了。」（不出现「不是来电不是短信」）
- **R-2**：把 4+ 微步**合并为 1–2 句**，中间必须插入**心理 / 环境 / 对话打断 / 失误**之一；允许保留**最多 3 个**连贯微动作，第 4 个起必须并句或换感受。
- **R-3**：禁止「又…又…。」后立刻接两个 ≤ 8 字的**纯状态验收短句**；若需保留，合并为一句带主观评价：「他点了几下，那行白字像钉在屏上。」

**检测信号**：

| 子维度 | PASS | WARN | 回滚级 FAIL |
|---|---|---|---|
| R-1 排除式枚举 | 全章 0 次「双否定 + 是/而是/只剩」整句式 | 1 次 | **≥ 2 次**；或 **1 次且与 G+1 的「不是 X 是 Y」模板同段叠加** |
| R-2 教程体链 | 任意连续 200 字内，无对白/无心理时，纯动作微步 ≤ 3 | = 4 | **≥ 5 个微步** |
| R-3 验收式对句 | 「又/再/然后」引导的动作块后，不接两个 ≤ 8 字独立验收句 | 接 1 个短验收句 | **接 ≥ 2 个 ≤ 8 字验收短句**（且中间无心理插入） |

**生成期规则**：
- 动笔前在内部提纲标注：**本章禁止出现「不是…不是…是」骨架**；若必须写否定，**同一段落最多 1 个否定词**指向该认知对象。
- 任何「点烟 / 解锁手机 / 翻包 / 付款码」类日常动作，**单段不得超过 3 个连贯微步**；超过必须插入半句走神、半句环境、或他人半句声。
- `webnovel-memory` 的 `chapter_meta.stats` 必须填报 `exclusion_enum_hits` / `tutorial_microstep_chain_max` / `catalog_afterthought_pairs`（R-3 命中次数）；`used-patterns.md` 滑窗追踪，上一章 R 任一 WARN → 下一章该对象动作强制改用「合并句 + 心理」写法。

### R-补充 · “不是X，是Y”再收紧（并入 R，不增加 25 项计数）

**症状**：虽未触发高频阈值，但在关键段反复使用“不是A，是B”做解释性对照，读者会立刻感到“模型在证明自己”。

**改写方向**：
- 关键段（章首 500 字、章尾 200 字、高潮前后 300 字）默认禁用“不是X，是Y”。
- 用“直接现象 + 角色误判”替代否定对照句。

**检测信号**：
- `contrastive_negation_hits`：全章“不是A，是B/而是B”命中
- `keyzone_contrastive_negation_hits`：关键段命中

**生成期规则**：
- `contrastive_negation_hits ≤ 1`，`>= 2` 判 FAIL。
- `keyzone_contrastive_negation_hits >= 1` 直接判 FAIL；首章命中按回滚级处理。

### R 与 G+1 的边界

- **G+1** 管的是**设定定义体**（剑骨 / 阵法 / 体系名）的「不是 X 是 Y」。
- **R-1** 管的是**日常叙事与器物描写**里的**双否定目录句**（来电 / 短信 / App 弹窗等）。两者都禁，但 R 专门堵「生活流里的说明书排比」——这是 G+1 未单独命名的实测高发区。

## 使用方式

### 生成 skill 调用本文件的流程

```
1. 读取本文件（或本文件的简表）
2. 生成前：把 A–R 共 18 主条款 + B/E/G/H/N 7 个子条款（共 25 项）全部作为硬约束写入 prompt
   · G-扩展 1 必须把 9 种"不是 X，是 Y"模板全文列入禁用清单
   · **R** 必须把「禁止双否定目录句」「单段微动作 ≤ 3」「显著时空跳变必须空行分段」写入 prompt
   · **A-补充** 必须把「章首 200 字内刺点钉子」「每 800–1200 字好奇缝隙」「禁止连续 6 段纯氛围」写入 prompt
   · **O-在场** 必须把「禁上一章/下一章/读者/作者/弹幕等元叙事词」写入 prompt，成稿全文检索
   · E 必须明文禁止"情绪词 + 粗体 / 独段"
   · N-细化 必须明文禁止"粗体主题句点题"
   · O 必须拉取本章 (a) 出场 ≥ 2 次有名角色 + (b) 首次登场关键角色 的 soul_fields 作为"必现清单"
   · D-2 必须预先为 ≥ 1 位配角指定"自主议题"
   · D-3 必须为选择/系统类机制预留"反直觉选项"
   · P-1 必须预先声明本章的"怪异预算"元素 ≥ 1 个（不可回收）
   · P-3 必须预先声明本章"延迟兑付"伏笔 ≥ 1 个（≥ 5 章内不回收）
   · P-4 必须先生成 5-清单并落盘到 `state/anti-trope-log.md`
   · **P-补充2** 必须把「共有文化符号贴进对抗动作」+「至少一处抬高预期后的拒展示/收束」写入 prompt，并在 `chapter_meta.stats` 填报 `cultural_shorthand_clash_hits` / `withhold_beat_present`
   · Q 必须对每次场景切换声明使用的桥类型与锚点，禁用转场词清单零容忍
   · 所有上一章的 used-patterns 命中项转化为本章禁用词
3. 生成中：每写完 500 字做一次 mini self-check（定义体 / 单句段连续 / 粗体 / 情绪独段 / 长段计数 / 转场桥声明）
4. 生成后：自查 18 主条款 + 7 子条款（共 25 项），命中任一 FAIL → 内部重写；回滚级 FAIL → 回退 plot-design 或 story-blueprint 对应步
5. 重写 2 轮仍命中 → 交付时连同命中项一起告诉用户
6. 若用户明确要求"就要 AI 爽文味"→ 允许豁免，但豁免项必须列出；D / O / P / Q / **R** / G-扩展 1 / N-细化 原则上不豁免
```

### 诊断 skill 调用本文件的流程

```
1. 读取用户文本
2. 对 A–R 共 18 主条款 + 7 子条款（共 25 项）逐项用检测信号判定：PASS / WARN / FAIL / 回滚级 FAIL
3. 输出命中项 + 最小改法
4. 对 FAIL / 回滚级 FAIL 项给原文摘录 + 改写示例
5. 回滚级 FAIL 必须说明"退回到哪个 workflow 的哪一步重做"
```

## 来源

本清单基于外部大模型（ChatGPT、Claude）对同一段 AI 生成网文的**多轮**联合诊断反推得出：

- **第 1–2 轮**：识别 A–L 共 12 项显性指纹
- **第 3 轮**（"已接近人类但仍 ~60% AI"的深度复查）：补充 B / E / G / H 的 4 条扩展子条款，以及 M（爽点链条过完整）与 N（质量曲线过稳定）两条元层级主条款
- **第 4 轮**（"修辞层已过关，但人都像纸做的"的主体性复查）：补充 O（角色灵魂缺位 / 功能性存在）——直接盯每个有名存在是不是一个主体，覆盖配角、反派、路人、动物、灵兽、法宝拟人
- **第 5 轮**（本 skill 自检出的第一章实测命中复查）：补充 / 收紧 —— G-扩展 1 加入 9 种"不是 X，是 Y"实测模板并给出分档阈值；E 加入"情绪词独段 / 粗体"禁令；E-扩展 2 加入 signature_reactions 明牌化禁令；G-细化 加入"角色 PPT 直讲设定"禁令；N-细化 加入"粗体主题句点题"禁令；C / K 增加明确量化阈值；O 修订阈值为"关键角色首次登场即生效、出场 ≥ 2 次即生效"
- **第 6 轮**（人类作家样本 vs 同 skill AI 生成样本的对比实测）：D 升级为回滚级并拆为 D-1/D-2/D-3 三个子维度（闲笔密度 / 配角自主议题 / 废选项戏剧化）；新增 **P** 剧情算法化 / 想象力贫血（含怪异预算 / 废选项 / 延迟兑付 / 反套路检查 4 子维度）；新增 **Q** 转场机械 / 黏滞转场（含 5 类桥 / 禁用转场词 / 摩擦点必填）；C 收紧为连续 ≤ 2、占比 ≤ 0.3；对比发现：人类文本的"真实感"大半来自 D + P + Q 三条而非修辞
- **第 7 轮**（签到流都市样本实测）：新增 **R** 说明书式排除枚举 + 教程体微动作链（回滚级，含 R-1/R-2/R-3）；**K-补充** 场景块空行分段（显著时空跳变必须双换行）与 Q、C 联动计数
- **第 8 轮**（连载元语言 + 平淡章首实测）：**O-在场** 禁「上一章/读者」等书籍体外坐标（人物须活在故事时间内）；**A-补充** 好奇缝隙与章首抓眼（小姨子范式：非常规关系 + 动作/声音 + 信息滞后）

凡是外部 LLM 用来"抓 AI"的特征，本清单都用来"反向规避"。
