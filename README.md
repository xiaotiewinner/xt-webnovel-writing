# xt-webnovel-writing

中文网络小说写作 agent —— 一个 OpenClaw / ClawHub skill。整个仓库就是一个 skill：根 `SKILL.md` 是唯一入口，内部由 6 个 `workflow.md` 模块组成完整的网文生成流水线，覆盖：参考文本拆解 → 立书 → 剧情设计 → 爽点与文笔 → 反 AI 味体检 → 长篇持久化记忆。

- 所有文案、模板、交付物默认**全中文**。
- 内置 **17 主条款 + 7 子条款（共 24 项）**「反 AI 味」硬约束，覆盖句式复用、主语熵、**单句段瀑布流（≤ 2 连续 & ≤ 0.3 占比）**、**信息密度与世界自主生活（含闲笔密度 / 配角自主议题 / 废选项戏剧化 · 回滚级）**、情感标签（含独段 + 粗体）、反应套餐、signature 明牌化、设定说明书、9 种定义体模板、角色 PPT 直讲、算法思维链、爽点链条过完整、质量曲线过稳定（含粗体主题句点题）、**角色灵魂缺位（含关键角色首登即生效）**、**剧情算法化 / 想象力贫血（含怪异预算 / 废选项 / 延迟兑付 / 反套路 5-清单 · 回滚级）**、**转场机械 / 黏滞转场（含 5 类桥 / 禁用转场词 / 摩擦点必填 · 回滚级）** 等"另一台模型才能抓出的深层指纹"。其中 D / M / N / N-细 / O / P / Q / G+1 / E（独段+粗体）/ K（0 长段）/ C（≥ 6 连续或占比 > 0.5）为**回滚级硬门**，任一命中强制整章退回指定 workflow 重写。
- **角色必有"灵魂"**：所有有名字的角色（含动物 / 灵兽 / 反派 / 路人）都必须有自己的核心创伤 / 私人欲望 / 私下信念 / 怪癖，并在出场时被"偶然看见"一次；不允许只为剧情功能存在的空心 NPC。
- 长篇场景（≥ 100 万字 / ≥ 200 章）有完整的磁盘持久化记忆系统，解决 AI 写长篇的失忆、设定漂移、伏笔遗忘、修为前后矛盾、**角色灵魂蒸发**等问题。

## 七条公理（全套共享）

1. 矛盾 → 剧情 → 爽点。倒挂就重写。
2. 主角第 1 章前 200 字内必须出场并产生动作。
3. 主角必有明显且持续生效的缺点。
4. 书名必须映射到一个具体的第一个爽点。
5. 反 AI 味 17 主条款 + 7 子条款（共 24 项）全部 PASS 才算生成成功；D / M / N / N-细 / O / P / Q / G+1 / E（独段+粗体）/ K（全章 0 长段）/ C（≥ 6 连续或占比 > 0.5）为**回滚级硬门**，任一命中整章回退指定 workflow 重写。
6. **角色灵魂硬门（分层）**：
   - 关键角色（主角 / POV / 女男主 / 核心反派 / 固定配角 top 5）：`soul_fields ≥ 2` 条，**首次登场章必须有 ≥ 1 处灵魂渗透（回滚级硬门）**
   - 重要有名角色（出场 ≥ 2 次）：`soul_fields ≥ 1` 条，每次出场至少 1 处渗透
   - 动物 / 灵兽需独立反应坐标轴；反派需第三维度
   - 灵魂渗透必须**非功能性**——删除该句后剧情推进不受影响才算数
7. 长篇（≥ 100 万字或 ≥ 200 章）必须走 memory · LOAD/PERSIST，绕过视为违规。
8. 短句 + 画面感 + 共鸣是文笔基线，辞藻华丽不是目标。

## 反 AI 味能力

针对"另一台大模型就能抓出来的 AI 味"，skill 内置 **17 主条款 + 7 子条款（共 24 项）**，并在生成阶段与诊断阶段双路执行。条款来源于外部大模型（ChatGPT / Claude）对同一段 AI 生成网文的**六轮联合诊断**反推得出（第 4 轮补了 O · 角色灵魂缺位；第 5 轮根据新章诊断细化出 G+1 的 9 种定义体模板、E 独段 + 粗体子条款、E+2 signature 明牌化、G-细 角色 PPT 直讲、N-细 粗体主题句点题、C/K 的回滚级阈值，以及 O 的"关键角色首登即生效"硬门；**第 6 轮对照人类高点击率章节诊断 AI 版本，升级 D → 世界自主生活 · 回滚级（含 D-1 闲笔 / D-2 配角议题 / D-3 废选项），新增 P · 剧情算法化 · 回滚级（含 P-1 怪异预算 / P-3 延迟兑付 / P-4 反套路 5-清单）与 Q · 转场机械 · 回滚级（5 类桥 / 禁用转场词 / 瞬移禁止 / 摩擦点必填），并进一步收紧 C 的单句段占比到 ≤ 0.3**）：

| # | 条款 | 生成期规避 | 诊断期信号 |
|---|---|---|---|
| A | 节拍器式推进 | 每 3–5 段插入无回报铺垫 | 连续 ≥ 5 段都可被打功能标签 |
| B | 句式与语义复用 | 重复句式 ≥ 2 次必换写 | 10 章内同句式 ≥ 10 次 |
| **B-扩展** | 主语单调 / 句长熵过低 | top-1 主语段占比 ≤ 40%，段长标准差 ≥ 8 | "他"开头段 ≥ 40% 且段长匀速 |
| **C** | "第 X 剑"节奏病 / 单句段瀑布流 | 最长连续单句段 ≤ 2；单句段占比 ≤ 0.3；**段落默认由"场景描写 + 动作 + 心理 + 对话" ≥ 2 类组合** | ≥ 6 连续单句段 或 占比 > 0.5 → **回滚级 FAIL** |
| **D** | 信息密度过"干净" / 世界没有自主生活（回滚级） | D-1 闲笔密度 ≥ 5 处 / 章 & 单处 ≥ 30 字；D-2 每章 ≥ 2 个配角自主议题（与主线无关）；D-3 废选项戏剧化 ≥ 2 处 | 每段都服务主线；配角 = 纸片；零与主线无关的日常摩擦 → **回滚级 FAIL** |
| E | 情感标签化 | 每形容词必须跟动作 / 生理反应 | 只有"冷静"没有证据 |
| **E-扩展** | 反应套餐化 | 反派卡必填 ≥ 2 条 signature；禁止"脸色/冷汗/胸口/沉默/'不可能'"标准套餐 | 反派段内标准套餐命中 ≥ 3 |
| F | 修为清单化 | 等阶嵌场景 + 代价，不独段 | 独立成段只写数字 |
| G | 设定说明书化 | 一次只透一条，通过体感带出 | "XX——YY。" 列表 |
| **G-扩展 1** | "是/不是/而是" 定义体 | 禁用定义排比；体验式 / 他人转述 / 残缺式 | 出现"不是 X，而是 Y"或"X，意味着——Y" |
| **G-扩展 2** | 三项并列打包讲解 | ≥ 3 项体系内容拆到 ≥ 3 章 | 同章讲体系 ≥ 3 项并列 |
| H | 最优解战斗 | 必植入干扰变量 + 战场外感官 | 全程完美无失误 |
| **H-扩展** | 算法式思维链 | 主角思考段必含 ≥ 1 条非理性噪声（误判/走神/情绪/杂念/非最优） | 思考段压成"观察→判断→调整→成功"四步无噪声 |
| I | 对话功能过载 | 换角色说同样行 → 重写 | 对话互换后无差别 |
| J | 套路节点密度过高 | 单章 ≤ 2 个类型节点 | 单章 ≥ 4 个节点 |
| K | 段长分布畸形 | 段长呈短/中/长阶梯 | 全章无 > 80 字段落 |
| L | 作者指纹缺位 | 强制 fingerprint.md 贴合 | 纯书面语无个人怪癖 |
| **M** | 爽点链条过完整 | 每条爽点必标 delay / denied / cost 打断点，当章兑现 | 章末 5 条爽点验证全 PASS 且无打断 |
| **N** | 质量曲线过稳定 | 每 2000–3000 字一次风格偏差；成稿后 1 段亮化 + 1 段回退粗化 | 全章找不到"明显差的一句"也找不到"明显亮的一句" |
| **O** | 角色灵魂缺位 / 功能性存在 | 关键角色 soul_fields ≥ 2 条 + 首登章 ≥ 1 处渗透（回滚级）；出场 ≥ 2 次的有名角色 soul_fields ≥ 1 条 + 每次出场 ≥ 1 处渗透；动物必填独立反应坐标轴；反派必填第三维度；渗透必须非功能性（deletion_verified） | 多数角色只剩功能；反派仅"想赢 / 恨主角"；动物纯工具化；配角台词可互换；**关键角色首登章无灵魂渗透 = 回滚级 FAIL** |
| **P** | 剧情算法化 / 想象力贫血（回滚级） | P-1 怪异预算 ≥ 2 处 / 章（剧情无法吸收的古怪细节）；P-3 延迟兑付 ≥ 1 处（当章抛出、≥ 3 章后才收）；P-4 写前预声明反套路 5-清单（常规下一步禁用），实写必偏离 ≥ 1 项 | 每条伏笔当章兑现；配角反应 100% 可预测；无任何"无法解释的怪异"；实写与套路 5-清单吻合 ≥ 4 → **回滚级 FAIL** |
| **Q** | 转场机械 / 黏滞转场（回滚级） | 每次转场必选 5 类桥之一（感官 / 物件 / 对话打断 / 摩擦点 / 情绪错位）；全章禁用"于是 / 接着 / 然后 / 紧接着 / 随后 / 下一秒 / 与此同时 / 时间一晃 / 不知不觉 / 一会儿后"等 10+ 转场词；每个新场景 ≥ 1 个摩擦点；禁止瞬移（场景跳跃必须带时间锚或物件桥） | 连续机械转场词 ≥ 3；新场景零摩擦；瞬间切场无桥 → **回滚级 FAIL** |

完整内容：`references/anti-ai-tells.md`。

**四条关键洞察**：
1. 最隐蔽的**语言层** AI 指纹是 **"太稳定地优秀"** —— 从头到尾维持同一质量曲线、主语匀速、段长匀速、情感反应模板化、爽点链条零打断、思考全程理性无噪声。M 和 N 两条就是为这个"元层级指纹"兜底的硬门。
2. 最隐蔽的**叙事层** AI 指纹是 **"人都像纸做的"** —— 每个角色只为剧情功能活着，没有属于自己的、与剧情无关的内部状态。动物只会预警救主，反派只有恨主角，路人甲可互换路人乙。O 条款就是为这个"主体性指纹"兜底的硬门；memory 模块会跨章监控每个有名角色的灵魂渗透缺位，连续 2 章缺位就把下一章该角色的灵魂渗透升级为回滚级硬门。
3. 最隐蔽的**世界层** AI 指纹是 **"世界没有自主生活"** —— 信息密度过于干净、配角没有与主角无关的议题、"废选项"不戏剧化。D 条款（回滚级）通过强制闲笔密度 / 配角自主议题 / 废选项戏剧化三道硬门，让世界"活过来"。
4. 最隐蔽的**剧情层** AI 指纹是 **"算法化推进 + 机械转场"** —— 所有伏笔当章兑现、反应 100% 可预测、转场用"于是/接着/紧接着"串起来。P 条款（回滚级）用"怪异预算 + 延迟兑付 + 反套路 5-清单"强制打破算法感；Q 条款（回滚级）用"5 类转场桥 + 禁用转场词 + 摩擦点必填"强制打破黏滞感。

## 长篇记忆系统（memory 模块）

目标规模：500 万字 / 6000 字每章 / ~830 章。

**项目目录契约**（SKILL.md §11，强制生效）：`project_root` 由 AI 依据用户意图唯一决定并锁定写入 `book.yaml`；**记忆、设定、章节等所有产出必须落在 `project_root` 下的固定子目录**；跨目录写入即 `PERSIST` 拒绝。

```
<project_root>/
├── book.yaml                  书籍元数据（含 project_root 锁定字段）
├── fingerprint.md             作者指纹（风格锁定）
├── bible/                     世界观圣经（静态设定 + 怪异预算池 world.md）
├── characters/                人物卡（一人一文件，含 soul_fields）
├── arcs/                      剧情弧
├── chapters/                  正文章节（每章独立文件）
├── state/                     动态状态：
│   ├── foreshadow.md          伏笔表（含延迟兑付标记，反 P-3）
│   ├── timeline.md            时间线
│   ├── power-level.md         修为等级
│   ├── used-excitement.md     已用爽点
│   ├── used-patterns.md       已用句式 / 段长熵 / 主语分布 / 闲笔密度 / 转场类型 / 怪异预算消耗
│   ├── anti-trope-log.md      反套路 5-清单（写前预声明 + 写后比对，反 P-4）
│   └── unresolved-conflicts.md 未收束矛盾
├── index/                     倒排索引（人物 / 地点 / 物品 / 章节摘要）
└── .webnovel-memory/          内部缓存（不可由用户直接编辑）
```

**工作流程**（每一章闭环）：

```
[用户]  写第 N 章
  ↓
[memory · LOAD]  拉出 ≤ 4800 字"记忆快照"：
  · book.yaml（含 project_root 契约） / fingerprint.md
  · 当前 arc 八步位置
  · 前 3 章摘要 + 第 N-1 章钩子
  · live 伏笔（top 3–5，含延迟兑付标记）
  · 活跃角色当前状态 + signature_reactions + **soul_fields（反 O 必现清单）**
  · 动物 / 灵兽独立反应坐标轴 + 上一章"纯功能性角色名单" / "动物纯工具化名单"
  · 禁用句式 / 禁用爽点类型 / 禁用反派套餐词 / **禁用转场词（反 Q）**
  · **世界自主生活监控**（闲笔密度 / 配角自主议题 / 废选项戏剧化，反 D）

  · **转场监控**（最近 5 章转场桥类型分布 / 禁用转场词命中，反 Q）
  · 反 AI 味监控面板摘要（主语分布 / 段长熵 / 单句段占比 / 非理性噪声覆盖率 / 质量方差 / 爽点打断类型 / 灵魂渗透缺位）
  ↓
[plot-design · draft_prose]  把快照 + anti-ai-tells + 反套路 5-清单预声明 全部嵌入 prompt
  ↓
[反 AI 味 17 主 + 7 子条款 自检（24 项）]  任一 FAIL 则重写；任一回滚级 FAIL（D / M / N / N-细 / O / P / Q / G+1 / E 独段+粗体 / K 0 长段 / C ≥6 连续或占比 > 0.5）强制整章退回指定 workflow 重写
  ↓
[memory · PERSIST]  8 步落盘（STEP 0 为路径契约校验）：
  · **STEP 0 · 路径契约校验**（所有写入路径必须匹配 project_root 下的固定子目录，否则拒绝）
  · chapters/<N>.md + frontmatter
  · index/chapter-log.md 摘要
  · state/foreshadow.md 伏笔表（含延迟兑付队列）
  · state/used-excitement.md / used-patterns.md / **state/anti-trope-log.md** 防重复 + 反套路日志
  · state/timeline.md / power-level.md
  · characters/<name>.md 每个在场角色的最新状态 + soul_bleed_log
  · arcs/<cur>.md 当前 arc 八步位置
  ↓
[一致性检查]  伏笔 / 修为 / 名词 / 爽点 / 场景 全部交叉验证
  ↓
[交付用户]
```

**周期性审计**：每写完 10 章，可要求跑 memory · AUDIT，自动发现：
- 超 100 章仍 live 的伏笔
- 无理由降级的修为
- 未入词典的专有名词
- 最近 20 章某单一爽点类型占比 > 40%
- 最近 20 章某单一打断类型占比 > 60%
- 最近 10 章全局主语 top-1 平均占比 > 35%
- 最近 10 章全局段长方差 / 具象度方差持续偏低（整书质量曲线过稳）
- 最近 10 章非理性噪声全局覆盖率 < 100%（整书算法味）
- 最近 10 章闲笔密度平均 < 5 处 / 章 或 配角自主议题覆盖率 < 60%（反 D 全书失守）
- 最近 20 章怪异预算池投放 < 2 处 / 章 或 延迟兑付队列持续为空（反 P 全书失守）
- 最近 10 章转场桥单一类型占比 > 60% 或 禁用转场词累计命中 > 0（反 Q 全书失守）
- 索引与章节 frontmatter 不一致
