# 立书 Agent · 静态设计稿生成

## 输入契约

启动前必须获取以下字段，缺字段用 `AskQuestion` 补齐：

```yaml
type: 玄幻 | 仙侠 | 都市 | 系统 | 重生 | 穿越 | 都市异能 | 悬疑 | ...
subgenre: 神豪 | 签到 | 四合院 | 年代重生 | 娱乐 | ...（可空）
core_pitch: <一句话梗概>
protagonist_hint: <主角身份的一句话，若用户没想好则允许为空，此时由 agent 生成 3 备选>
target_length: 短篇（<5 万字）| 中篇（5–30 万字）| 长篇（>30 万字）
mood: 爽 | 黑深残 | 治愈 | 悲情 | 诙谐
```

## 管线（必须按顺序执行，不允许跳步）

```
STEP 1 主线矛盾链
STEP 2 人设（主角 + 1–2 长期配角）
STEP 3 书名（11 类起名法）
STEP 4 大纲骨架
STEP 5 简介
STEP 6 作者指纹（下游生成正文时必须按此固化风格）
```

前一步的产出是下一步的输入。用户若只要其中一步，依然需要先在内部推导前置步骤，再只输出用户要的那步。

---

## STEP 1 · 主线矛盾链

### 动作
1. 把 `core_pitch` 改写为 **矛盾句**：`[主角] × [对手/阻力] × [为了 X] × [阻力形式]`。
2. 把该矛盾拆成 L1–L5 递进层，填下表。
3. 每层必须能答出"解决后世界变成什么样"。答不出 → 合并或删层，再补一层。
4. 从 L1 倒推 **切入点事件**（故事开始时对主角造成巨大影响的那件事）。

### 输出 schema

```markdown
## 主线矛盾链

- 矛盾句：[主角] × [对手/阻力] × [为了 X] × [阻力形式]
- 切入点事件：<一句话，指明故事第 1 章开场主角正遭遇什么>

| 层级 | 主角位置 | 对手 / 压迫源 | 解决后世界变成什么样 |
|---|---|---|---|
| L1 身份 | | | |
| L2 实力 | | | |
| L3 圈子 / 宗派 / 组织 | | | |
| L4 领域 / 大陆 / 行业 | | | |
| L5 换地图 / 规则跃迁 | | | |
```

### 拒绝条件
- 矛盾句里有"奇幻之旅""非凡人生"此类空话 → 重写。
- 某一层"解决后"没有可观察的世界变化 → 重写。

## STEP 2 · 人设

### 动作
1. 根据矛盾句 + `protagonist_hint` 生成主角表。
2. **必须给主角一个能被矛盾链持续使用的缺点**（胆小 / 心软 / 贪财 / 不信任人 / 身体缺陷 / 过度谨慎 ……）。
3. 生成 1–2 个长期配角，每个配角先标注"服务主角哪项功能"（情报 / 武力 / 情感支点 / 对照组），再立即补第 4 步。
4. **灵魂字段强制输出（反 AI 味 O 条款回滚级硬门）**：按角色分层硬性要求：
   - **关键角色**（主角 / POV / 女男主 / 核心反派 / 固定配角 top 5）：**≥ 2 条 soul_fields**
   - **重要有名角色**（预估总出场 ≥ 2 次的有名角色）：**≥ 1 条 soul_fields**
   - 角色是动物 / 灵兽 / 灵物 → 必须填独立反应坐标轴 ≥ 1 条
   - 角色是反派 → 必须填第三维度 ≥ 1 条
   - **任一分层不达标 → 整个 STEP 2 重写，不允许跳过**
5. 若有长期出场的动物 / 灵兽 / 法宝拟人，也当作角色生成一张卡。


### 拒绝条件
- 主角"完美无缺" → 重写。
- 配角的存在不能落到"服务主角的哪一项" → 删除或重写。
- **灵魂字段空缺或被填成"围绕主角 / 为主角赴死 / 忠心 / 崇拜主角" → 整张人物卡判定 AI 味 O 失败，必须重写**。
- 反派的第三维度仍然指向"想赢 / 恨主角 / 要权力" → 重写。
- 动物 / 灵兽的独立反应坐标轴仍在服务主角指令 → 重写。

## STEP 3 · 书名（11 类起名法）

### 11 类定义

| # | 类型 | 代表 | 使用条件 | 风险 |
|---|---|---|---|---|
| 1 | 人名型 | 《张三丰异界游》《悟空传》 | 用公众熟知的名字（同人/历史名人） | 陌生人名 → 无记忆点 |
| 2 | 职业型 | 《武神》《药王》《极品家丁》 | 职业词汇大众熟知 | 生造职业 → 无期待 |
| 3 | 技能型 | 《诛仙》《斗破苍穹》 | 已成 IP 可借势 | 新作过于笼统 |
| 4 | 年代型 | 《回到明朝当王爷》《重生1998》 | 历史/年代重生 | 模糊年代 → 失去锚 |
| 5 | 地点型 | 《斗罗大陆》 | 蹭大神热度 | 指向性不明 |
| 6 | 主线型 | 《七界武神》《异界之极品奶爸》 | 一句话表达主线 | 受众自我收窄 |
| 7 | 分类型 | 《重生之……》《四合院之……》 | 流派文标配 | 局限强 |
| 8 | 期待型 | 《史上第一妖》《鬼吹灯》《宋朝没事》 | **最推荐** | 起名难度高 |
| 9 | 历程型 | 《凡人修仙传》 | 线性成长叙事 | 传统感强 |
| 10 | 东方型 | 《莽荒纪》《佛本是道》 | 东方玄幻神话 | 当下偏旧 |
| 11 | 反衬型 | 《平平无奇大师兄》《师兄太稳健了》 | 有独特噱头 | 过滤读者 |

### 动作
1. 根据 `type + subgenre + core_pitch`，从 11 类里**挑 3 类**最可能高效的。
2. 每类生成 1 个候选 → 共 3 候选。
3. 对每个候选立即标注：**对应的第一个爽点是什么**（读者看到这个名字会立刻期待什么）。
4. **严禁**输出没有第一个爽点映射的书名。

### 输出 schema

```markdown
## 书名候选

| 候选 | 所属类型 | 第一个爽点（读者看名字后立刻期待的事） | 风险提示 |
|---|---|---|---|
| 《 》 | 第 X 类：名称 | | |
| 《 》 | 第 X 类：名称 | | |
| 《 》 | 第 X 类：名称 | | |

## 推荐
- 首选：<候选 X>
- 理由：<为什么它和主线 + 类型最贴>
```

## STEP 6 · 作者指纹

生成这本书**专属的 2–4 条写作怪癖**，下游所有正文生成必须一致沿用。没有指纹 → 正文读起来就是"工业 AI 味"（见 `../references/anti-ai-tells.md` § L）。

### 字段

```markdown
## 作者指纹 · 《书名》

| 维度 | 本作设定 | 反例（禁止的表达） |
|---|---|---|
| 偏好动词（2–4 个） | 例："挑 / 扯 / 丢 / 晾" | 禁用："淡淡地说 / 缓缓地" |
| 偏好物象（1–3 个） | 例：烟 / 旧书 / 某一类光 | — |
| 主角口头禅或标志反应 | 例：开口前先"嗯"一声；紧张时抠指节 | 禁用："深吸一口气" |
| 断句偏好 | 例：短逗号串 / 破折号嵌套 / 短问句结尾 | 禁用：四字成语连缀 |
| 叙事人称与视角 | 例：第三人称限知，只进主角视角 | 禁用：全知切视角 |
| 章末钩子偏好 | 例：用一个动作 + 一句半截话结尾 | 禁用："未完待续"式收束 |
```

### 拒绝条件
- 任一维度写成"风格鲜明 / 具有个性"等空话 → 重写到能复现的具体行为。
- 与反 AI 味清单 §E（情感标签）/ §G（说明书设定）直接冲突的偏好 → 改写。

## STEP 5 · 简介

### 四要素（缺一不可）
背景 + 主要人物形象 + 冲突 + 悬念。

### 结构模板

```markdown
[第 1 段：背景 + 立刻的冲突（1–2 句）]
[第 2 段：主角处境升级，拉出次级冲突（1–2 句）]
[第 3 段：期待钩子（一句反问 / 一句自嘲 / 一句大话）]
[第 4 段：与书名呼应的爽点点睛（1 句）]
```

### 参考样本
> 刚穿越到玄幻世界，就遭遇气运之子即将打脸的情况？顾长歌表示有点懵。
>
> 不仅女主被她圣主老爹亲自送到自己身边，就连和气运之子许下三年之约的前未婚妻，都对自己无比仰慕？
>
> 啧，开局就拉满了气运之子的仇恨值。
>
> 这感觉，有些酸爽啊。
>
> 好在自己的身份有点牛逼，是上界不朽大教下来历练的真传弟子。
>
> 踩死一个小小的气运之子，这还不简单？
>
> 等等，还有个专搞主角的系统？
>
> 看来这是要自己在天命大反派的路上越走越远啊。

### 自检
- [ ] 第 1 句包含背景
- [ ] 出现主角 + ≥ 1 个有名有姓配角
- [ ] 有一句"等等 / 啧 / 可是 / 然而…"的翻折
- [ ] 最后一句与书名互为镜像
- [ ] 整段没有"踏上征程""揭开序幕"等空话

## 落盘（长篇场景）

当用户目标是长篇（`target_length == 长篇` 或明确说"500 万字 / 写系列"等）：

1. 按 SKILL.md §11 确认 / 锁定 `project_root`（用户显式指定 > `book.yaml.project_root` > 用户给的短名在当前工作目录建目录）。
2. 调用 `webnovel-memory` · INIT 初始化**全部固定子目录**（`book.yaml` / `fingerprint.md` / `bible/` / `characters/` / `arcs/` / `chapters/` / `state/`（含 `anti-trope-log.md`）/ `index/` / `.webnovel-memory/`）。
3. 把本 skill 的输出字段一对一灌入（**所有写入路径严格按 SKILL.md §11 契约**，禁止在 `project_root` 外或契约外子目录写文件）：
   - STEP 1 主线 → `<project_root>/book.yaml.main_axis` + `book.yaml.l1_conflict`；同时写入 `book.yaml.project_root`（绝对 / 相对一致）锁定
   - STEP 2 人设 → 每个角色一个 `<project_root>/characters/<name>.md`，主角标 `role: 主角`；**灵魂字段**一对一落入 `characters/<name>.md` 的 `## 灵魂字段（soul_fields）` 段。按分层硬门校验：
     - 关键角色（主角 / POV / 女男主 / 核心反派 / 固定配角 top 5）`soul_fields ≥ 2` 条
     - 重要角色 `soul_fields ≥ 1` 条
     - 反派含 `third_dimension`；动物 / 灵兽含独立反应坐标轴
     - 任一不达标 → 拒绝落盘并回到 STEP 2 补齐（反 O 回滚级硬门）
   - STEP 3 书名 → `book.yaml.title` + `book.yaml.first_excitement`
   - STEP 4 大纲 → 按 L1–L5 层级分解为多个 `<project_root>/arcs/arc-<NN>-<slug>.md`（每 arc 一行矛盾句 + 八步位置占位）
   - STEP 5 简介 → `book.yaml.synopsis`
   - STEP 6 作者指纹 → `<project_root>/fingerprint.md`（完整 6 字段）
   - 若 STEP 2 涉及世界观要素（修为体系 / 势力 / 术语）→ 分别写入 `<project_root>/bible/power-system.md` / `bible/factions.md` / `bible/glossary.md`
   - **STEP 7 世界观怪异预算池（反 P-1 · 必填）**：在 `<project_root>/bible/world.md` 末尾生成 `## 怪异预算池` 段，填入 ≥ 10 条候选"剧情无法吸收"的细节 / 场景 / 角色 / 风俗（例："城北街尾每逢雨夜会出现一个吹口哨的小孩，没人知道他家在哪。" 例："灵兽受惊会把舌头吐出来贴在主人手背上。"）。下游 `webnovel-plot-design` 每章必抽 ≥ 1 条落入正文，5 章内不得回收。池子剩余 ≤ 3 条 → 提示补充。
4. 成功后提示用户："项目已固化到 {project_root}，后续续写请走 webnovel-plot-design 的 draft_prose 模式。"

短中篇不需要落盘，直接返回档案。
