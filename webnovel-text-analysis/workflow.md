# 参考文本拆解 Agent

## 触发条件

仅当用户**主动提供**以下任一作为输入时启动：

- 整章或多章文本
- 一段范文（500 字以上）
- 书名列表（≥ 3 个）
- 简介样本
- 明确命名的作品 + 用户已粘贴其内容

**若用户只报作品名但没贴文本**：不要凭记忆杜撰内容；让用户粘贴文本，或告知 agent 不做联网检索。

## 输入契约

在开始前必须收到或追问以下字段：

```yaml
material_type: chapter | excerpt | title_list | synopsis | mixed
material: |
  <原文内容>
analysis_goal: extract_pattern | imitate_style | evaluate_quality | find_weakness | other
user_own_project: |
  <用户自己在写的作品一句话；缺失则填 N/A>
```

缺字段时先用 `AskQuestion` 补齐，不要硬猜。

## 四种拆解模式

根据 `material_type` 选模式。每种模式的输出 schema 固定，不允许变体。

### 模式 A · 章节拆（material_type = chapter / excerpt）

#### A.1 整章字段表

```markdown
## [章节标题/序号] 字段表

| 字段 | 内容 |
|---|---|
| 类型 / 流派 | |
| 切入点事件（故事从什么事件开始） | |
| 主角登场字数（第几字） | |
| 主角人设（一句话 + 缺点） | |
| 第一个梗（作品类型决定） | |
| 本章主要矛盾 | |
| 本章小高潮位置（第几字） | |
| 本章结尾钩子（一句话） | |
| 期待感来源（读者被什么问题勾住） | |
```

#### A.2 前 1000 字按字段段拆

只对第 1 章或第 1 段样本执行：

```markdown
## 字段段拆

- 前 200 字：交代了 [ ]；出场人物 [ ]；暗示冲突 [ ]
- 前 500 字：加入 [ ]；小矛盾点 [ ]
- 前 1000 字：小剧情收束 / 悬念抛出 [ ]
- 结尾钩子：[ ]
```

#### A.3 可复用资产

```markdown
## 可复用资产

### 梗 · [一句话命名]
- 铺垫（前文在哪里埋）：
- 爆发（本段怎么爆）：
- 升华（爽点放大方式）：
- 承接（余波如何导向下一段）：
- 可迁移到用户作品的地方：[对照 user_own_project]

### 描写模板 · [打扮 / 人物 / 景色 / 动作 / 声音 之一]
- 原句：
- 泛化模板：[把专有名词替换成 <槽位>]

### 句式 · [短句串 / 倒装 / 白描 / 共鸣句]
- 原句：
- 使用条件：
```

若文本里没有某种资产，**明确写 "未检出"，不要编造**。

### 模式 B · 风格模仿（analysis_goal = imitate_style）

额外产出一份 **风格指纹**：

```markdown
## 风格指纹

- 平均句长：X 字（统计估算）
- 短句占比：高 / 中 / 低
- 倒装 / 省略 / 口语化：命中哪几项
- 对话比例：高 / 中 / 低
- 描写密度：画面感多 / 只写动作 / 只写情绪
- 常用动词：[列 5 个]
- 叙事人称：第一 / 第三 限知 / 第三 全知
- 爽点类型分布：武力 / 桃花 / 信息差 / 身份优越 / ……（按出现次数排）
- 章节钩子偏好：对话悬念 / 外部事件 / 内心独白 / 反问 / ……
```

然后在用户确认之前不生成任何模仿文本，只交付指纹。

### 模式 C · 书名列表归类（material_type = title_list）

把每个书名归到 11 类起名法中的哪一类：

```markdown
## 书名归类

| 书名 | 类型 | 推断的第一个爽点 |
|---|---|---|
| … | 1 人名 / 2 职业 / 3 技能 / 4 年代 / 5 地点 / 6 主线 / 7 分类 / 8 期待 / 9 历程 / 10 东方 / 11 反衬 | … |

## 规律提取
- 最高频类型：
- 该列表的命名共性（词频 / 句式）：
- 用户若要起一个同质书名：[给 3 个候选]
```

11 类定义以 `webnovel-story-blueprint` 为准。

### 模式 D · 质量诊断（analysis_goal = evaluate_quality / find_weakness）

同 `webnovel-pitfalls` 协作：先跑模式 A 的字段表，然后把 A 的结果传给十大误区自检。不要两个 skill 同时跑两份独立诊断。

## 输出前的强制检查

- [ ] 每个字段都有内容或"未检出"（不得留空）。
- [ ] 没有引用任何用户未提供的素材。
- [ ] 若用户提供了 `user_own_project`，资产表的"可迁移到用户作品的地方"字段必须填。

## 与其他 skill 的交接

- 用户让你"按这个风格写"之后：把风格指纹传给 `webnovel-plot-design` / `webnovel-excitement-and-craft` 作为生成约束。
- 用户让你"学这个梗用在我的书上"：把梗提取表传给 `webnovel-plot-design`。
